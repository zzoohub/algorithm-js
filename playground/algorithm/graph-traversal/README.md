# Graph Traversal (그래프 탐색)

## BFS (Breadth-First Search, 너비 우선 탐색)

### 동작 원리
시작 정점에서 **가까운 것부터** 레벨 단위로 탐색한다.
**큐**를 사용한다.

```
그래프:
  A --- B --- E
  |     |
  C --- D --- F

BFS(A): A → B → C → D → E → F
(거리 0) (거리 1) (거리 2)
```

### 핵심 코드 흐름
1. 시작 정점을 큐에 넣고 방문 표시
2. 큐에서 꺼내서 이웃을 확인
3. 방문 안 한 이웃을 큐에 넣고 방문 표시
4. 큐가 빌 때까지 반복

### 특성
- 시간: O(V + E)
- 공간: O(V)
- **최단 경로 보장** (가중치 없는 그래프에서)

---

## DFS (Depth-First Search, 깊이 우선 탐색)

### 동작 원리
한 방향으로 **끝까지** 깊이 들어간 후 되돌아온다.
**재귀** 또는 **스택**을 사용한다.

```
DFS(A): A → B → D → C → F → E
(한 방향으로 끝까지, 그 다음 방향)
```

### 재귀 vs 반복 (스택)
- **재귀**: 코드가 간결. 단, 깊은 그래프에서 스택 오버플로우 위험
- **스택**: 명시적 제어. 재귀 깊이 제한 없음

### 특성
- 시간: O(V + E)
- 공간: O(V)
- 최단 경로 보장 안 함

## 사이클 감지

### 3-color 알고리즘 (방향 그래프)
- **WHITE (미방문)**: 아직 탐색 안 함
- **GRAY (탐색 중)**: 현재 DFS 경로에 포함
- **BLACK (완료)**: 탐색 완료, 더 이상 볼 필요 없음

GRAY 노드의 이웃이 GRAY → 현재 경로에서 다시 만남 → **사이클**

## BFS vs DFS 언제 쓰나?

| 상황 | BFS | DFS |
|------|-----|-----|
| 최단 경로 | O | X |
| 모든 경로 탐색 | X | O |
| 메모리 (넓은 그래프) | 많이 사용 | 적게 사용 |
| 사이클 감지 | 가능 | 더 자연스러움 |
| 위상 정렬 | Kahn 알고리즘 | 후위 순서 역순 |

## 실무에서의 활용

### BFS
- **소셜 네트워크**: "친구의 친구" 추천 (2-hop BFS)
- **웹 크롤러**: 시작 URL에서 링크를 따라가며 페이지 수집
- **최단 경로**: 네비게이션, 미로 탈출
- **레벨 순서 처리**: 트리의 레벨별 출력

### DFS
- **순환 의존성 감지**: npm, webpack 모듈 그래프에서 circular dependency 발견
- **위상 정렬**: 빌드 순서, 작업 스케줄링
- **미로 생성**: 랜덤 DFS로 미로를 생성
- **강결합 컴포넌트 (SCC)**: 그래프의 강결합 요소 찾기 (Tarjan 알고리즘)
- **백트래킹**: 스도쿠, N-Queen 등 조합 탐색
