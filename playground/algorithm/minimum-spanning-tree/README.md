# Minimum Spanning Tree (최소 신장 트리)

## 개념

그래프의 **모든 정점을 연결**하면서 **간선 가중치의 합이 최소**인 트리.
V개의 정점을 정확히 V-1개의 간선으로, 사이클 없이, 비용을 최소화한다.

```
원래 그래프:                  MST:
  A ──4── B                   A ──4── B
  |╲     ╱|                   |       |
  2  3  1  5                  2       1
  |   ╲╱   |                  |       |
  D ──6── C                   D       C

총 비용: 4+2+1 = 7 (최소!)
```

## 왜 "트리"인가?

- V개의 정점을 연결하는 최소 간선 수는 V-1
- V-1개의 간선으로 V개 정점을 연결하면 반드시 트리 (사이클 없는 연결 그래프)
- 간선을 더 빼면 연결이 끊기고, 더 넣으면 사이클이 생김

## 두 가지 알고리즘

### Kruskal — 간선 중심 (전역적 그리디)

**"가장 싼 간선부터 사이클 안 만들면 추가"**

```
간선 정렬: (B,C,1), (A,D,2), (A,B,4), (A,C,3), (B,D,5), (D,C,6)

1. (B,C,1) → B와 C가 다른 그룹 → 추가 ✓  {B,C}
2. (A,D,2) → A와 D가 다른 그룹 → 추가 ✓  {A,D} {B,C}
3. (A,C,3) → A와 C가 다른 그룹 → 추가 ✓  {A,B,C,D} ← 3개 간선, 완성!
4. (A,B,4) → A와 B가 같은 그룹 → 사이클! 건너뜀 ✗
```

**Union-Find의 대표적 활용 사례.** `find()`로 같은 컴포넌트인지 O(α(n))에 판별, `union()`으로 합침.

### Prim — 정점 중심 (지역적 그리디)

**"현재 트리에서 가장 가까운 정점을 하나씩 추가"**

```
시작: {A}

1. A에서 가장 가까운 간선: A-D(2) → D 추가, MST = {A,D}
2. {A,D}에서 가장 가까운: A-C(3) → C 추가, MST = {A,C,D}
3. {A,C,D}에서 가장 가까운: B-C(1) → B 추가, MST = {A,B,C,D} 완성!
```

Dijkstra와 거의 동일한 구조. 차이는 우선순위 기준:
- Dijkstra: "출발점에서의 **총** 거리"
- Prim: "현재 트리까지의 **직접** 연결 비용"

## Kruskal vs Prim

| | Kruskal | Prim |
|---|---|---|
| 접근 | 간선 중심 | 정점 중심 |
| 자료구조 | **Union-Find** | **Min Heap** |
| 시간 | O(E log E) | O(E log V) |
| 간선이 적을 때 (sparse) | **더 효율적** | - |
| 간선이 많을 때 (dense) | - | **더 효율적** |

## 그리디가 최적인 이유: 교환 논증 (Exchange Argument)

MST 알고리즘은 **그리디(탐욕법)**인데, 왜 항상 최적해를 보장하는가?

```
귀류법: Kruskal이 선택한 간선 e가 최적 MST에 없다고 가정.
→ 최적 MST에 e를 추가하면 사이클이 생김.
→ 그 사이클에서 e보다 무거운 간선 e'가 반드시 존재.
   (안 그러면 Kruskal이 e를 먼저 선택했을 리가 없으므로)
→ e'를 빼고 e를 넣으면 더 가벼운 MST가 됨 → 모순!
```

이 **교환 논증(exchange argument)**은 그리디 알고리즘의 정당성을 증명하는 핵심 기법이다.

## 시간 복잡도

| 알고리즘 | 시간 | 공간 |
|---------|------|------|
| Kruskal | O(E log E) | O(V + E) |
| Prim (Heap) | O(E log V) | O(V + E) |
| Prim (인접 행렬) | O(V²) | O(V²) |

## 실무에서의 활용

### 1. 네트워크 설계
도시 간 광케이블, 전력망, 수도관 설치 비용 최소화.
모든 도시를 연결하되 총 건설 비용을 최소로.

### 2. 클러스터링 (Single-Linkage Clustering)
MST를 구한 후 가장 무거운 간선부터 제거하면 클러스터가 형성.
K개의 클러스터를 원하면 K-1개의 가장 무거운 간선을 자르면 된다.
→ 이미지 세그멘테이션, 유전자 분류에 사용.

### 3. 근사 알고리즘의 기반
TSP(외판원 문제)의 2-근사 알고리즘: MST의 가중치 × 2 ≥ 최적 TSP 비용.
NP-hard 문제의 근사해를 MST로 빠르게 구함.

### 4. 회로 설계
VLSI 칩 설계에서 핀들을 최소 와이어로 연결 (Steiner Tree 문제의 근사).

### 5. 왜 알아야 하는가
MST는 **그리디 알고리즘이 최적인 대표적 사례**이고, 그 증명(교환 논증)은 "왜 이 그리디가 되는가?"를 판별하는 범용 도구다. 이 사고방식은 허프만 코딩, 활동 선택 문제 등 다른 그리디 문제의 정당성 증명에도 그대로 적용된다.
