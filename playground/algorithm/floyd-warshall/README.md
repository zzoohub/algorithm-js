# Floyd-Warshall 알고리즘

## 개념

**모든 정점 쌍 (i, j)의 최단 거리**를 한 번에 구하는 알고리즘.
3중 루프 하나로 O(V³)에 끝난다. 음수 가중치도 처리 가능.

```
Dijkstra: "서울에서 다른 모든 도시까지의 최단 거리"  → 단일 출발점
Floyd-Warshall: "모든 도시 쌍의 최단 거리"           → 모든 쌍
```

## 핵심 DP 점화식

```
dp[k][i][j] = "정점 0, 1, ..., k를 중간 경유지로 사용할 수 있을 때,
               i에서 j까지의 최단 거리"

dp[k][i][j] = min(
  dp[k-1][i][j],                    ← k를 안 거침
  dp[k-1][i][k] + dp[k-1][k][j]    ← k를 거침: i→k→j
)
```

### 점화식의 직관

"경유할 수 있는 정점의 집합을 하나씩 늘려간다."

```
k=0: 직통 간선만 사용 가능
     A → B: 10,  A → C: ∞,  B → C: 3

k=1: 정점 A를 경유 가능
     (여기선 A를 경유해서 개선되는 게 없음)

k=2: 정점 A, B를 경유 가능
     A → C: min(∞, 10 + 3) = 13  ← A→B→C로 개선!

k=V: 모든 정점을 경유 가능 = 최종 최단 거리
```

## 동작 예시

```
초기 그래프 (인접 행렬):
     A    B    C
A  [ 0    3    ∞ ]
B  [ ∞    0    1 ]
C  [ 2    ∞    0 ]

k=0 (A를 경유):
  B→C: min(1, ∞+∞) = 1     (변화 없음)
  C→B: min(∞, 2+3) = 5     ← C→A→B!

k=1 (A,B를 경유):
  A→C: min(∞, 3+1) = 4     ← A→B→C!
  C→A: min(2, 5+∞) = 2     (변화 없음)

k=2 (A,B,C를 경유):
  A→B: min(3, 4+∞) = 3     (변화 없음)
  B→A: min(∞, 1+2) = 3     ← B→C→A!

최종:
     A    B    C
A  [ 0    3    4 ]
B  [ 3    0    1 ]
C  [ 2    5    0 ]
```

## 음수 사이클 검출

Floyd-Warshall이 끝난 후 **dist[i][i] < 0**인 정점이 있으면 음수 사이클이 존재.
(자기 자신으로 돌아오는데 비용이 음수 = 무한히 돌면 비용이 -∞)

```
dist[A][A] = -2  → A를 포함하는 음수 사이클 존재
```

## Dijkstra vs Bellman-Ford vs Floyd-Warshall

| | Dijkstra | Bellman-Ford | Floyd-Warshall |
|---|---|---|---|
| 목적 | 단일 출발 → 나머지 | 단일 출발 → 나머지 | **모든 쌍** |
| 음수 가중치 | 불가 | 가능 | **가능** |
| 음수 사이클 검출 | 불가 | 가능 | **가능** |
| 시간 | O((V+E) log V) | O(VE) | **O(V³)** |
| 모든 쌍 구할 때 | O(V(V+E) log V) | O(V²E) | **O(V³)** |

V가 작고 (수백 이하) 모든 쌍이 필요하면 Floyd-Warshall이 구현도 간단하고 효율적.

## 시간 복잡도

| | 복잡도 |
|---|---|
| 시간 | O(V³) |
| 공간 | O(V²) |
| 경로 복원 | O(V) per path |

## 실무에서의 활용

### 1. 네트워크 라우팅 (OSPF/BGP)
소규모 네트워크에서 모든 라우터 쌍의 최단 경로를 미리 계산.
라우팅 테이블을 한 번에 구축할 수 있다.

### 2. 도달 가능성 분석 (Transitive Closure)
가중치를 무시하고 "i에서 j에 도달 가능한가?"만 판별.
Floyd-Warshall을 boolean 연산으로 바꾸면 O(V³)에 전이적 폐포를 구할 수 있다.
→ 의존성 분석, 접근 제어 검증에 사용.

### 3. 그래프 지름 (Diameter) 계산
모든 쌍 최단 거리를 구한 후 최댓값 = 그래프의 지름.
SNS의 "최대 몇 단계로 연결되는가?" (Six Degrees of Separation).

### 4. 도시 간 경로 최적화
항공권, 물류 등에서 "모든 도시 쌍의 최적 경로"를 미리 계산해두고
사용자 질의에 O(V) 경로 복원으로 즉시 응답.

### 5. 왜 DP 사고의 정수인가
Floyd-Warshall의 상태 설계 — "사용할 수 있는 자원(정점)을 하나씩 늘려간다" — 는 배낭 문제, 비트마스크 DP 등 많은 DP 문제에서 반복되는 패턴이다. 이 사고방식을 체화하면 DP 문제의 상태 정의가 자연스러워진다.
