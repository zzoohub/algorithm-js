# Dynamic Programming (동적 프로그래밍)

## 개념

**큰 문제를 작은 부분 문제로 나누고, 부분 문제의 결과를 저장해서 재사용**하는 기법.
같은 계산을 반복하지 않는다.

### DP가 적용 가능한 조건
1. **최적 부분 구조**: 큰 문제의 최적해가 작은 문제의 최적해로 구성됨
2. **중복 부분 문제**: 같은 작은 문제가 반복적으로 나타남

### Top-down vs Bottom-up
```
Top-down (메모이제이션):
  큰 문제 → 작은 문제로 분해 (재귀) + 캐시
  fib(5) → fib(4) + fib(3) → ... (이미 계산한 건 캐시에서 꺼냄)

Bottom-up (타뷸레이션):
  작은 문제부터 순서대로 채워나감 (반복문)
  fib(0)=0, fib(1)=1, fib(2)=1, fib(3)=2, fib(4)=3, fib(5)=5
```

## 구현된 패턴들

### 1. 피보나치 — DP 입문
- 재귀만 쓰면 O(2^n) → 메모이제이션 적용하면 O(n)
- 공간 최적화: 이전 두 값만 유지하면 O(1) 공간

### 2. 0/1 배낭 (Knapsack)
```
아이템:  무게 [2, 3, 4, 5], 가치 [3, 4, 5, 6], 용량 8
dp[i][w] = i번째까지 고려, 용량 w에서 최대 가치

각 아이템마다: 넣는다 vs 안 넣는다
dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])
```

### 3. LCS (최장 공통 부분수열)
```
a = "ABCBDAB", b = "BDCAB"
LCS = "BCAB" (길이 4)

같은 문자면: dp[i][j] = dp[i-1][j-1] + 1
다르면:      dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

### 4. 동전 교환 (Coin Change)
```
동전 [1, 5, 10], 금액 13
dp[0]=0, dp[1]=1, ..., dp[5]=1, ..., dp[10]=1, dp[11]=2, dp[12]=3, dp[13]=4
→ 10+1+1+1 = 4개
```

### 5. LIS (최장 증가 부분수열)
```
[10, 9, 2, 5, 3, 7, 101, 18]
LIS = [2, 3, 7, 18] → 길이 4

이진 탐색 최적화로 O(n log n)
```

---

## DP 문제 접근법

1. **점화식 세우기**: dp[i]가 무엇을 의미하는지 정의
2. **기저 조건**: dp[0], dp[1] 등 초기값 설정
3. **전이 관계**: dp[i]를 이전 값들로 어떻게 계산하는지
4. **순서**: 어떤 순서로 채워야 하는지
5. **정답 위치**: dp의 어디에 최종 답이 있는지

## 실무에서의 활용

### 1. 텍스트 diff (Git, 에디터)
Git의 diff 알고리즘이 LCS 기반이다.
두 파일의 공통 부분을 찾고, 다른 부분을 +/- 로 표시.

### 2. 스펠 체커 (Edit Distance)
"kitten" → "sitting"의 최소 편집 횟수 = 3 (DP로 계산).
검색 엔진의 "이것을 찾으셨나요?" 기능.

### 3. 리소스 배분
클라우드 서버에 작업을 배분할 때 (비용/성능 트레이드오프) → 배낭 문제.

### 4. DNA 서열 비교
생물정보학에서 DNA 서열의 유사도를 LCS/Edit Distance로 계산.

### 5. 경로 최적화
최소 비용으로 격자를 이동하는 문제, 로봇 이동 경로 계획.

### 6. 텍스트 줄바꿈
TeX/LaTeX의 줄바꿈 알고리즘이 DP로 최적의 줄바꿈 위치를 결정.
