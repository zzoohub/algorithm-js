# Topological Sort (위상 정렬)

## 개념

**DAG (Directed Acyclic Graph, 방향 비순환 그래프)**에서
의존 관계를 만족하는 순서로 정점을 나열하는 것.

"A를 해야 B를 할 수 있다"는 관계에서, 올바른 실행 순서를 구한다.

```
수학 → 물리 → 양자역학
수학 → 통계
영어 → 논문작성

위상 정렬: [영어, 수학, 논문작성, 물리, 통계, 양자역학]
(여러 결과가 가능)
```

## 시간 복잡도

| 방법 | 시간 |
|------|------|
| Kahn (BFS) | O(V + E) |
| DFS | O(V + E) |

---

## Kahn's Algorithm (BFS 기반)

### 동작 원리
1. 모든 정점의 **진입 차수 (in-degree)** 계산
2. 진입 차수가 0인 정점을 큐에 넣음 (의존성이 없는 것)
3. 큐에서 꺼내서 결과에 추가, 이웃의 진입 차수를 1 감소
4. 진입 차수가 0이 된 이웃을 큐에 넣음
5. 큐가 빌 때까지 반복
6. 결과에 모든 정점이 포함되지 않으면 → **사이클 존재**

```
A → B → D
A → C → D

진입 차수: A=0, B=1, C=1, D=2
1. A 처리 → B=0, C=0
2. B 처리 → D=1
3. C 처리 → D=0
4. D 처리

결과: [A, B, C, D]
```

---

## DFS 기반

### 동작 원리
1. 모든 정점에 대해 DFS 수행
2. DFS가 끝나는 시점(후위 순서)에 결과에 추가
3. 결과를 뒤집으면 위상 순서
4. 탐색 중인 노드를 다시 만나면 → **사이클**

## 실무에서의 활용

### 1. 빌드 시스템
webpack, rollup의 모듈 번들링:
```
index.js → utils.js → logger.js
index.js → api.js → utils.js
```
위상 정렬로 어떤 파일부터 빌드할지 결정.

### 2. 패키지 매니저
`npm install` 시 의존성 트리를 위상 정렬해서 설치 순서 결정.
순환 의존성이 있으면 경고.

### 3. 작업 스케줄링
CI/CD 파이프라인에서 작업 간 의존 관계:
```
lint → test → build → deploy
        ↓
      coverage
```

### 4. 스프레드시트 셀 계산 순서
A1 = B1 + C1, B1 = D1 * 2 일 때, D1 → B1 → A1 순서로 계산.
순환 참조가 있으면 에러.

### 5. 데이터베이스 마이그레이션
마이그레이션 스크립트 간 의존 관계를 정렬해서 실행 순서 결정.

### 6. React useEffect / useMemo 의존성
React 내부적으로 컴포넌트 렌더 순서를 결정할 때 의존성 그래프를 활용.
