# Bit Manipulation (비트 연산)

## 개념

컴퓨터의 **가장 기본적인 연산 단위**인 비트를 직접 조작하는 기법.
모든 데이터는 결국 0과 1의 비트이다.

```
10진수  7 = 이진수 0111
10진수 12 = 이진수 1100
```

## 기본 연산자

| 연산 | 기호 | 설명 | 예시 (5=101, 3=011) |
|------|------|------|------|
| AND | `&` | 둘 다 1이면 1 | 101 & 011 = 001 (1) |
| OR | `\|` | 하나라도 1이면 1 | 101 \| 011 = 111 (7) |
| XOR | `^` | 다르면 1 | 101 ^ 011 = 110 (6) |
| NOT | `~` | 반전 | ~101 = ...010 |
| 왼쪽 시프트 | `<<` | 비트를 왼쪽으로 | 1 << 3 = 1000 (8) |
| 오른쪽 시프트 | `>>` | 비트를 오른쪽으로 | 8 >> 2 = 10 (2) |

## 핵심 트릭들

### 1. n & (n-1) — 가장 낮은 1비트 끄기
```
12 = 1100
11 = 1011
 &  = 1000    ← 가장 오른쪽 1이 사라짐
```
용도: 2의 거듭제곱 판별, 1의 개수 세기

### 2. n & (-n) — 가장 낮은 1비트만 추출
```
12  = 001100
-12 = 110100 (2의 보수)
 &  = 000100   ← 가장 낮은 1비트만 남음
```
용도: Fenwick Tree(BIT)의 핵심 연산

### 3. XOR의 성질
- `a ^ a = 0` (같은 수 XOR = 0)
- `a ^ 0 = a` (0과 XOR = 자기 자신)
- 교환법칙, 결합법칙 성립

배열에서 하나만 홀수 번 나타나는 수 찾기:
```
[2, 3, 2, 4, 3] → 2^3^2^4^3 = (2^2)^(3^3)^4 = 0^0^4 = 4
```

## 비트마스크 — 집합을 정수로 표현

원소 {a, b, c, d}가 있을 때, 부분집합을 4비트 정수로 표현:
```
{a, c}    = 0101 (5)
{b, c, d} = 1110 (14)
{a, b, c, d} = 1111 (15)
{}        = 0000 (0)
```

### 집합 연산이 비트 연산으로 O(1)
- 합집합: `A | B`
- 교집합: `A & B`
- 차집합: `A & ~B`
- 원소 추가: `A | (1 << i)`
- 원소 제거: `A & ~(1 << i)`
- 원소 확인: `A & (1 << i)`

## 실무에서의 활용

### 1. Unix 파일 권한
```
rwxr-xr-- = 111 101 100 = 754
chmod 755 = rwxr-xr-x
```
`(perm & WRITE) !== 0` → 쓰기 권한이 있는가?

### 2. Feature Flags / 권한 시스템
```ts
const ADMIN = 0b0001;
const EDITOR = 0b0010;
const VIEWER = 0b0100;
user.role = ADMIN | EDITOR; // 0b0011
```
정수 하나로 여러 역할을 표현, DB에 정수 하나만 저장.

### 3. React의 Fiber Flags
React 내부에서 컴포넌트의 상태를 비트 플래그로 관리:
```
Placement | Update | Deletion = 0b111
```

### 4. 네트워크 서브넷 마스크
```
IP:   192.168.1.100 = 11000000.10101000.00000001.01100100
Mask: 255.255.255.0 = 11111111.11111111.11111111.00000000
AND:  192.168.1.0   ← 네트워크 주소
```

### 5. 해시 함수
해시 함수가 비트 연산(XOR, 시프트)을 사용해서 입력을 섞는다.
FNV-1a, MurmurHash 등이 비트 연산 기반.

### 6. 게임 개발
체스 보드를 64비트 정수로 표현 (Bitboard).
각 비트가 한 칸의 말의 존재를 나타냄.
