# KMP (Knuth-Morris-Pratt) 문자열 매칭

## 개념

텍스트에서 패턴을 **O(n+m)**에 찾는 알고리즘.
나이브 매칭이 불일치마다 처음부터 다시 비교하는 것과 달리,
KMP는 **"이미 일치한 부분의 정보를 재활용"**해서 텍스트 포인터를 절대 되돌리지 않는다.

```
나이브 매칭:
  텍스트: A B C A B C A B D
  패턴:   A B C A B D         ← D에서 불일치
          ↓ 한 칸 밀고 처음부터 다시!
            A B C A B D       ← A부터 다시 비교 (낭비!)

KMP:
  텍스트: A B C A B C A B D
  패턴:   A B C A B D         ← D에서 불일치
              ↓ "AB"가 이미 일치했으니 거기서부터!
              A B C A B D     ← C부터 비교 (AB는 건너뜀!)
```

## 핵심: Failure Function (실패 함수)

### 정의
`failure[i]` = 패턴의 `[0..i]` 부분에서 **진접두사(proper prefix) = 접미사**인 최대 길이.

"진접두사"란 문자열 전체를 제외한 접두사.

### 계산 예시
```
패턴: A B A B A C

i=0: "A"        → 진접두사: (없음)     → failure[0] = 0
i=1: "AB"       → 진접두사: A ≠ 접미사 B  → failure[1] = 0
i=2: "ABA"      → A = A (접두 A = 접미 A) → failure[2] = 1
i=3: "ABAB"     → AB = AB              → failure[3] = 2
i=4: "ABABA"    → ABA = ABA            → failure[4] = 3
i=5: "ABABAC"   → 접두사=접미사 없음    → failure[5] = 0

failure = [0, 0, 1, 2, 3, 0]
```

### 왜 이게 중요한가?

```
텍스트: A B A B A B A C
패턴:   A B A B A C         ← C에서 불일치 (i=5까지 일치)

failure[4] = 3이므로:
"ABABA"의 접두사 "ABA" = 접미사 "ABA"

즉, 텍스트의 현재 위치 앞 3글자 "ABA"는
패턴의 처음 3글자 "ABA"와 이미 일치하는 상태!

→ 패턴을 한 번에 2칸 밀어서 j=3부터 비교

텍스트: A B A B A B A C
패턴:       A B A B A C     ← index 3(B)부터 비교!
```

## 동작 과정

```
텍스트:  A A B A A B A A B
패턴:    A A B A A B
failure: [0,1,0,1,2,3]

i=0: A=A, j=1
i=1: A=A, j=2
i=2: B=B, j=3
i=3: A=A, j=4
i=4: A=A, j=5
i=5: B=B, j=6 → j==m → 매치! 위치 0

이제 j = failure[5] = 3 (다음 매치를 찾기 위해)

i=6: A=A, j=4
i=7: A=A, j=5
i=8: B=B, j=6 → j==m → 매치! 위치 3

결과: [0, 3]에서 패턴 발견
```

## 나이브 vs KMP

| | 나이브 | KMP |
|---|---|---|
| 시간 | O(n × m) | **O(n + m)** |
| 텍스트 포인터 | 되돌아감 | **절대 안 돌아감** |
| 전처리 | 없음 | failure 함수 O(m) |

## Rabin-Karp (롤링 해시)

다른 접근법: 패턴과 텍스트 윈도우의 **해시값**을 비교.

```
패턴 "ABC"의 해시 = 123
텍스트: X A B C D E
        ↓ 윈도우
       [X A B] hash = 456 ≠ 123 → 다음
         [A B C] hash = 123 = 123! → 실제 비교 → 매치!
```

**롤링 해시**: 윈도우를 한 칸 밀 때 O(1)에 해시 갱신.
```
hash("ABC") = hash("XAB") - X의 기여 + C의 기여
```

KMP보다 구현이 간단하고, **여러 패턴을 동시에 검색**할 때 강점.

## 시간 복잡도

| 알고리즘 | 전처리 | 검색 | 최악 |
|---------|--------|------|------|
| 나이브 | 없음 | O(nm) | O(nm) |
| **KMP** | O(m) | **O(n)** | **O(n+m)** |
| Rabin-Karp | O(m) | O(n) 평균 | O(nm) 최악 |

## 실무에서의 활용

### 1. 텍스트 에디터의 Find/Replace
Ctrl+F로 문자열을 찾을 때. 대용량 파일에서 빠른 검색이 필수.
VS Code, Vim의 내부 검색 엔진.

### 2. 컴파일러 / 렉서 (Lexer)
소스 코드에서 키워드, 식별자, 리터럴을 인식하는 과정.
패턴 매칭이 컴파일러의 첫 단계(lexical analysis).

### 3. 바이러스/침입 탐지 시스템 (IDS)
네트워크 패킷에서 악성 시그니처 패턴을 실시간 탐색.
Snort, Suricata 같은 IDS가 문자열 매칭 알고리즘을 핵심으로 사용.

### 4. DNA 서열 분석
"이 유전자 서열에 특정 패턴이 나타나는가?" — 수십억 글자의 게놈에서 검색.
실제로는 KMP의 확장인 Aho-Corasick(다중 패턴)을 많이 사용.

### 5. 왜 알아야 하는가
KMP의 failure 함수는 **"부분적 성공을 재활용한다"**는 통찰의 결정체.
이 사고방식은 DP, suffix array, 오토마타 이론과 직결되며, "정보를 버리지 않고 최대한 활용한다"는 알고리즘 설계의 근본 원칙을 체화하게 해준다.
