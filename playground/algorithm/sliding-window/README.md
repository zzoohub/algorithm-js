# Sliding Window & Two Pointers (슬라이딩 윈도우 & 투 포인터)

## 개념

배열/문자열에서 **연속된 구간(윈도우)**을 효율적으로 탐색하는 기법.
중첩 반복문 O(n²)을 O(n)으로 줄인다.

```
고정 크기 윈도우 (k=3):
[1, 3, 2, 6, -1, 4, 1, 8]
 [-----]                    합 = 6
    [-----]                 합 = 11
       [-----]              합 = 7
          ...
→ 앞에서 하나 빼고 뒤에서 하나 더하면 O(1)에 업데이트
```

---

## 패턴 1: 고정 크기 슬라이딩 윈도우

윈도우 크기 k가 정해져 있을 때.

### 동작 원리
1. 처음 k개의 합/상태를 계산
2. 윈도우를 한 칸 밀 때마다: `새 원소 추가 - 빠지는 원소 제거`
3. 각 위치에서 결과를 업데이트

### 예: 크기 3 부분배열의 최대합
```
[2, 1, 5, 1, 3, 2]  k=3
초기: 2+1+5 = 8
이동: 8-2+1 = 7
이동: 7-1+3 = 9  ← 최대
이동: 9-5+2 = 6
```

---

## 패턴 2: 가변 크기 슬라이딩 윈도우

조건을 만족하는 최소/최대 윈도우를 찾을 때.

### 동작 원리
1. right를 오른쪽으로 확장
2. 조건을 만족하면 left를 줄여본다 (더 좋은 답이 있을까?)
3. 조건이 깨지면 다시 right를 확장

### 예: 합이 7 이상인 최소 길이
```
[2, 3, 1, 2, 4, 3]  target=7
→ [4, 3] (길이 2)
```

---

## 패턴 3: 투 포인터

정렬된 배열에서 두 포인터가 양끝에서 시작해 조건에 따라 이동.

### 예: 두 수의 합
```
[1, 2, 4, 6, 10]  target=8
left=0(1), right=4(10) → 11 > 8 → right--
left=0(1), right=3(6)  → 7 < 8  → left++
left=1(2), right=3(6)  → 8 = 8  ✓
```

---

## 패턴 4: 슬라이딩 윈도우 + 자료구조

윈도우 내의 최대/최소를 유지하려면 **Deque**를 결합한다.

### 슬라이딩 윈도우 최대값
Deque의 앞쪽에 항상 현재 윈도우의 최대값 인덱스를 유지:
- 새 원소가 들어올 때: 뒤에서부터 새 원소보다 작은 것들 제거
- 윈도우를 벗어난 인덱스: 앞에서 제거
- 시간: O(n) — 각 원소가 최대 한 번 추가, 한 번 제거

## 실무에서의 활용

### 1. 실시간 모니터링
"최근 5분간의 평균 응답 시간" → 고정 크기 슬라이딩 윈도우.
Prometheus, Grafana가 이 방식으로 메트릭을 계산.

### 2. Rate Limiting
"최근 1분간 API 요청이 100회를 초과하면 차단" → 슬라이딩 윈도우 카운터.
고정 윈도우(분 단위 리셋)보다 정확하다.

### 3. 네트워크 TCP 흐름 제어
TCP의 Sliding Window Protocol: 수신자가 "이만큼 더 보내도 돼"라고
윈도우 크기를 알려준다.

### 4. 스트리밍 데이터 분석
실시간 로그에서 "최근 N개 이벤트 중 에러 비율" 계산.

### 5. DNA 서열 분석
고정 길이 k-mer를 슬라이딩 윈도우로 추출.

### 6. 이동 평균 (Moving Average)
주식 차트의 이동 평균선, 센서 데이터의 노이즈 필터링.
