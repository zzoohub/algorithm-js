# A* Search (A-스타 탐색)

## 개념

**"Dijkstra + 목적지 방향 힌트"** = 최적 경로를 보장하면서 탐색 범위를 줄이는 알고리즘.
Dijkstra가 모든 방향으로 균등하게 퍼져나가는 반면, A*는 목적지 쪽으로 집중한다.

```
핵심 공식: f(n) = g(n) + h(n)

g(n): 시작점 → n까지 실제로 걸린 비용 (확정)
h(n): n → 목적지까지 얼마나 걸릴지 추정 (휴리스틱)
f(n): "이 노드를 거쳐가면 총 비용이 얼마일까?" (추정)

→ f(n)이 가장 작은 노드부터 탐색
```

## 시각적 비교

```
S = 시작, G = 목표, # = 벽, · = 탐색한 칸

BFS/Dijkstra:                A*:
· · · · · · ·               · · ·
· · · S · · ·               · S · ·
· · · · # · ·                 · · #
· · · · · # · · G               · · # · G
· · · · · · ·                     · ·
· · · · ·
탐색 노드: 35개               탐색 노드: 12개
```

같은 최단 경로를 찾지만, A*가 훨씬 적은 노드를 탐색한다.

## 휴리스틱 함수

### 그리드에서의 대표적 휴리스틱

```
현재 위치 (2,3), 목적지 (5,7)

Manhattan 거리: |5-2| + |7-3| = 7   (상하좌우만 이동 가능할 때)
Euclidean 거리: √((5-2)² + (7-3)²) = 5.0  (대각선 이동 가능할 때)
```

### Admissibility (허용 가능성) 조건

**h(n)이 실제 거리를 절대 과대 추정하지 않으면**, A*는 최단 경로를 보장한다.

```
실제 거리가 10일 때:
  h(n) = 8   → admissible ✓ (과소 추정 OK)
  h(n) = 10  → admissible ✓ (정확 OK)
  h(n) = 12  → NOT admissible ✗ (과대 추정 → 최적성 깨질 수 있음)
```

- Manhattan 거리: 상하좌우 이동에서 admissible (대각선이 없으니 정확하거나 과소)
- Euclidean 거리: 항상 admissible (직선 = 가능한 최단 거리)
- h(n) = 0: 항상 admissible → Dijkstra와 동일 (정보 없음)

**h(n)이 실제에 가까울수록** 탐색 효율이 좋다 (탐색 노드가 적다).
**h(n) = 0**이면 모든 방향으로 탐색 → Dijkstra.
**h(n) = 실제 거리**이면 최단 경로를 일직선으로 찾음 (이상적이지만 구할 수 없으니 추정).

## A*와 Dijkstra의 관계

```
A*:       f(n) = g(n) + h(n)
Dijkstra: f(n) = g(n) + 0      ← h(n) = 0인 A*
Greedy:   f(n) = 0 + h(n)      ← g(n) = 0인 A* (최적 보장 안 됨)
```

A*는 Dijkstra의 일반화이고, 휴리스틱의 품질로 성능이 결정된다.

## 동작 예시

```
그리드:    S . . .
           . # # .
           . . . G

시작 S(0,0), 목표 G(2,3), Manhattan 휴리스틱

Open: [S(f=5)]  // g=0, h=5

1. S(0,0) 선택, f=5
   이웃: (0,1) g=1,h=4,f=5 | (1,0) g=1,h=4,f=5
   Open: [(0,1,f=5), (1,0,f=5)]

2. (0,1) 선택, f=5
   이웃: (0,2) g=2,h=3,f=5
   Open: [(1,0,f=5), (0,2,f=5)]

3. (0,2) 선택, f=5
   이웃: (0,3) g=3,h=2,f=5
   ...

4-6. 계속 진행...

최종 경로: S → (0,1) → (0,2) → (0,3) → (1,3) → G
비용: 5
```

## 시간 복잡도

| | 최악 | 좋은 휴리스틱 |
|---|---|---|
| 시간 | O(E log V) (Dijkstra와 동일) | 실제로는 훨씬 빠름 |
| 공간 | O(V) | O(V) |

최악의 경우(h=0)는 Dijkstra와 같지만, 좋은 휴리스틱이 있으면 탐색 노드가 지수적으로 줄어든다.

## 실무에서의 활용

### 1. 게임 AI (경로 탐색)
RTS 게임, RPG의 유닛 이동. StarCraft, Age of Empires 등 거의 모든 게임이 A* 변형을 사용.
대규모 맵에서는 HPA*(Hierarchical Pathfinding A*)로 확장.

### 2. 지도/내비게이션
Google Maps, 카카오내비 등의 경로 탐색.
실제로는 A* + Contraction Hierarchies 같은 전처리 기법을 결합.

### 3. 로봇 경로 계획
자율주행차, 드론의 경로 계획. 장애물 회피하면서 목적지까지 최적 경로.
ROS(Robot Operating System)의 기본 경로 계획 알고리즘.

### 4. 퍼즐 솔버
15-퍼즐, 루빅스 큐브 등의 최적해 탐색.
상태 공간을 그래프로 모델링하고 A*로 최소 이동 횟수 탐색.

### 5. 왜 알아야 하는가
A*의 핵심 통찰 — **"정보를 활용해 탐색 공간을 줄인다"** — 는 CS 전반에 적용되는 원리다. 데이터베이스 쿼리 최적화(통계로 실행 계획 선택), 컴파일러 최적화(휴리스틱으로 레지스터 할당), 기계학습(탐색 공간 가지치기)이 모두 같은 사고 구조를 공유한다.
