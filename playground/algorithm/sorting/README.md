# Sorting Algorithms (정렬 알고리즘)

## 비교 정렬의 한계
비교 기반 정렬은 이론적으로 **O(n log n)**보다 빠를 수 없다 (결정 트리 하한).

---

## Merge Sort (병합 정렬)

### 동작 원리
1. 배열을 반으로 나눈다 (분할)
2. 각 반쪽을 재귀적으로 정렬한다 (정복)
3. 정렬된 두 반쪽을 합친다 (병합)

```
[38, 27, 43, 3, 9, 82, 10]
         ↓ 분할
[38, 27, 43]   [3, 9, 82, 10]
    ↓ 분할          ↓ 분할
[38] [27, 43]  [3, 9] [82, 10]
       ↓              ...
[38] [27] [43] → 병합 → [27, 38, 43]
                → 최종 병합 → [3, 9, 10, 27, 38, 43, 82]
```

### 특성
| | |
|---|---|
| 시간 | 항상 O(n log n) — 최악에도 보장 |
| 공간 | O(n) — 추가 배열 필요 |
| 안정 정렬 | O (같은 값의 원래 순서 유지) |

### 실무
- Python의 `sorted()`, Java의 `Arrays.sort(Object[])`가 TimSort (Merge Sort 변형)
- 외부 정렬 (디스크에 있는 거대한 파일 정렬)에 적합

---

## Quick Sort (퀵 정렬)

### 동작 원리
1. **피벗**을 하나 선택한다
2. 피벗보다 작은 것은 왼쪽, 큰 것은 오른쪽으로 분류 (파티션)
3. 왼쪽과 오른쪽을 재귀적으로 정렬

```
피벗=4: [3, 6, 8, 10, 1, 2, 1]
         → [3, 1, 2, 1] [4] [6, 8, 10]
         → 재귀...
```

### 특성
| | |
|---|---|
| 시간 (평균) | O(n log n) |
| 시간 (최악) | O(n²) — 이미 정렬된 배열 + 나쁜 피벗 |
| 공간 | O(log n) — in-place |
| 안정 정렬 | X |

### Median of Three
최악을 피하기 위해 첫/중간/마지막 값의 중간값을 피벗으로 선택.
이 구현에서 사용하는 방식.

### 실무
- C의 `qsort()`, V8의 `Array.sort()`가 Quick Sort 변형을 사용
- 캐시 효율이 Merge Sort보다 좋아서 실제로는 더 빠른 경우가 많음

---

## Counting Sort (계수 정렬)

### 동작 원리
값의 범위가 제한적일 때, **각 값의 출현 횟수를 세서** 정렬.
비교를 하지 않는다.

```
입력: [4, 2, 2, 8, 3, 3, 1]
카운트: [0, 1, 2, 2, 1, 0, 0, 0, 1]  (인덱스 = 값)
누적합: [0, 1, 3, 5, 6, 6, 6, 6, 7]
출력:   [1, 2, 2, 3, 3, 4, 8]
```

### 특성
| | |
|---|---|
| 시간 | O(n + k) — k는 값의 범위 |
| 공간 | O(n + k) |
| 안정 정렬 | O |
| 제한 | 정수 또는 값의 범위가 작을 때만 효율적 |

### 실무
- Radix Sort의 서브루틴으로 사용
- 문자열의 문자 빈도 세기 (아나그램 검사)
- 히스토그램 생성

---

## 비교 요약

| 알고리즘 | 평균 | 최악 | 공간 | 안정 | 특징 |
|----------|------|------|------|------|------|
| Merge Sort | O(n log n) | O(n log n) | O(n) | O | 항상 보장, 외부 정렬 |
| Quick Sort | O(n log n) | O(n²) | O(log n) | X | 실전에서 가장 빠름 |
| Counting Sort | O(n+k) | O(n+k) | O(n+k) | O | 범위가 작을 때 |
