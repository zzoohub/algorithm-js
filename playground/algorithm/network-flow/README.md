# Network Flow (네트워크 유량)

## 개념

**"파이프 네트워크에서 수원(source)에서 배수구(sink)로 최대 얼마나 흘릴 수 있는가?"**

각 파이프(간선)에 용량이 있을 때, source에서 sink로의 최대 유량을 구한다.

```
용량:  S ──10──→ A ──5──→ T
       |          ↑        ↑
       └──8──→ B ──3──→ ──┘
                 └──7──────┘

최대 유량 = 15 (S→A→T: 5, S→B→T: 7, S→A←B→T: 3)
```

## 핵심 정리: Max-Flow = Min-Cut

**최대 유량 = 네트워크를 S쪽과 T쪽으로 자르는 최소 비용.**

이것은 조합론에서 가장 강력한 이중성(duality) 중 하나다.

```
Min-Cut: S와 T를 분리하는 최소 비용 절단

     S쪽    |    T쪽
  S ──10──→ | A ──5──→ T
  |         |  ↑        ↑
  └──8──→ B | ──3──→ ──┘
             |──7──────┘
         cut=10+8=18? No...

실제 min-cut:
  S ──10──→ A │──5──→ T
  |           │ ↑        
  └──8──→ B  │──3──→ T
              │──7──→ T
  cut = 5+3+7 = 15 = max flow!
```

## 알고리즘: Edmonds-Karp (BFS 기반 Ford-Fulkerson)

### Ford-Fulkerson 방법
1. **증가 경로(augmenting path)** 찾기: S→T로 잔여 용량이 있는 경로
2. 경로의 **병목(bottleneck)** = 경로 상 최소 잔여 용량
3. 그만큼 유량을 보내고 잔여 용량 갱신
4. 더 이상 증가 경로가 없으면 종료

### 잔여 그래프 (Residual Graph)와 역방향 간선

이것이 가장 비직관적이면서 핵심적인 부분:

```
용량 10인 간선에 7의 유량이 흐르면:

원래 간선:   u ──10──→ v   (잔여 용량: 10-7 = 3)
역방향 간선: u ←──7──  v   (유량을 "취소"할 수 있는 가상 간선)
```

**왜 역방향 간선이 필요한가?**
처음에 잘못된 경로로 유량을 보냈더라도, 역방향 간선을 통해 "유량을 되돌리고" 더 좋은 경로로 재배분할 수 있다. 이것이 없으면 지역 최적에 갇힌다.

```
예) S→A→T: 1, S→B→T: 0 에서 멈추면 최적이 아님
    역방향 간선으로 S→B→A(역방향)→T를 찾으면 유량 재배분
    → S→A→T: 1, S→B→T: 1 로 개선!
```

### Edmonds-Karp = BFS로 증가 경로 탐색
BFS를 쓰면 가장 짧은 증가 경로를 찾으므로 **O(VE²)**에 수렴이 보장된다.

## 동작 예시

```
    S ──4──→ A ──3──→ T
    |        ↑↓       ↑
    └──3──→ B ──5──→─┘

1단계: BFS로 S→A→T (병목=3), 유량 3
    잔여: S→A:1, A→T:0, S→B:3, B→T:5, A↔B

2단계: BFS로 S→B→T (병목=3), 유량 3
    잔여: S→A:1, S→B:0, B→T:2

3단계: BFS로 S→A→B→T (병목=1), 유량 1
    잔여: S→A:0, A→B가 줄고, B→T:1

4단계: 더 이상 경로 없음

최대 유량 = 3 + 3 + 1 = 7
```

## 이분 매칭 (Bipartite Matching)

Network Flow의 대표적 응용. 두 그룹을 1:1로 연결하는 최대 매칭.

```
지원자: {A, B, C}     직무: {1, 2, 3}
A → 1, 2 지원
B → 2, 3 지원
C → 1 지원

가상 네트워크:
S →1→ A →1→ 1 →1→ T
S →1→ B →1→ 2 →1→ T
S →1→ C →1→ 3 →1→ T

최대 유량 = 3 → 최대 매칭 = 3
A→2, B→3, C→1
```

## 시간 복잡도

| 알고리즘 | 시간 |
|---------|------|
| Ford-Fulkerson (DFS) | O(E × maxFlow) |
| **Edmonds-Karp (BFS)** | **O(VE²)** |
| Dinic | O(V²E) |

## 실무에서의 활용

### 1. 이분 매칭: 작업 배정
직원-프로젝트 배정, 서버-요청 할당, 강사-수업 배정.
"각 직원이 할 수 있는 프로젝트 목록이 있을 때 최대 몇 개를 동시에 진행할 수 있는가?"

### 2. 이미지 세그멘테이션
전경(foreground)과 배경(background)을 분리.
각 픽셀을 S(전경) 또는 T(배경)에 할당, 인접 픽셀 간 유사도를 간선 용량으로.
Min-cut = 가장 자연스러운 경계선.

### 3. 스포츠 리그 탈락 판정
"이 팀이 수학적으로 우승 가능성이 있는가?"
남은 경기를 유량 문제로 모델링해서 판별. MLB에서 실제로 사용된 역사적 사례.

### 4. 네트워크 안정성 분석
"이 네트워크에서 몇 개의 간선/노드를 끊으면 S와 T가 분리되는가?"
Min-cut = 네트워크의 취약점 분석.

### 5. 왜 알아야 하는가
Max-Flow/Min-Cut은 **"이 문제가 사실은 유량 문제다"**라고 인식하는 순간 사용할 수 있다. 겉보기에 전혀 다른 문제들 — 매칭, 스케줄링, 이미지 처리, 네트워크 분석 — 이 유량 모델로 환원되는 것을 보면 "문제를 변환하는" CS적 사고가 깊어진다.
