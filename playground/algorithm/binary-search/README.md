# Binary Search (이진 탐색)

## 개념

**정렬된 배열**에서 탐색 범위를 **매번 절반으로 줄여**서 O(log n)에 찾는다.
1000만 개의 데이터에서 최대 24번 비교만으로 찾을 수 있다.

```
[1, 3, 5, 7, 9, 11, 13, 15] target=9

1. lo=0, hi=7, mid=3 → arr[3]=7 < 9 → lo=4
2. lo=4, hi=7, mid=5 → arr[5]=11 > 9 → hi=4
3. lo=4, hi=4, mid=4 → arr[4]=9 ✓ 찾음!
```

## 시간 복잡도

| | |
|---|---|
| 시간 | O(log n) |
| 공간 | O(1) |

---

## 변형들

### Lower Bound (target 이상인 첫 위치)
```
[1, 2, 4, 4, 4, 7] target=4
                ↑ 인덱스 2
```
"4가 처음 나오는 위치" 또는 "4를 삽입할 가장 왼쪽 위치"

### Upper Bound (target 초과인 첫 위치)
```
[1, 2, 4, 4, 4, 7] target=4
                         ↑ 인덱스 5
```
"4보다 큰 첫 위치" → upperBound - lowerBound = 해당 값의 개수

### lo < hi vs lo <= hi
- `lo <= hi`: **정확한 값** 찾기. 못 찾으면 -1.
- `lo < hi`: **경계** 찾기 (lower/upper bound). lo와 hi가 만나는 지점이 답.

### mid 계산
```ts
// 이렇게 하면 lo + hi가 정수 범위를 넘을 수 있다 (C/C++/Java)
const mid = Math.floor((lo + hi) / 2);

// 안전한 방법
const mid = lo + Math.floor((hi - lo) / 2);
// 또는
const mid = lo + ((hi - lo) >> 1);
```

## Parametric Search (매개변수 탐색)

"조건을 만족하는 최적의 값을 찾아라" → 이진 탐색으로 변환

### 패턴
1. **답의 범위** [lo, hi]를 설정
2. mid로 **조건을 검증** (O(n) 등)
3. 조건 만족 → 더 좋은 답이 있나? (범위 축소)
4. 조건 불만족 → 반대쪽으로 이동

### 예시: 나무 자르기
```
나무 높이: [20, 15, 10, 17]
최소 M 미터의 나무를 가져가려면 절단 높이 H의 최대값은?

H=15: 잘린 양 = 5+0+0+2 = 7
H=13: 잘린 양 = 7+2+0+4 = 13
→ 이진 탐색으로 최적의 H를 찾는다
```

## 실무에서의 활용

### 1. 데이터베이스 B-Tree 인덱스
DB의 인덱스 검색이 이진 탐색의 확장이다.
`WHERE id = 42` → B-Tree에서 이진 탐색으로 O(log n)에 찾음.

### 2. Git bisect
"이 버그가 어느 커밋에서 생겼나?" → 커밋 히스토리를 이진 탐색.
```
git bisect start
git bisect bad     # 현재 커밋은 버그 있음
git bisect good v1.0  # 이 버전은 정상
→ 중간 커밋을 체크아웃해서 확인 반복
```

### 3. 시스템 설정 최적화
"서버 몇 대를 투입해야 응답시간이 100ms 이하가 되는가?"
→ 서버 수를 이진 탐색, 각 mid에서 벤치마크 실행

### 4. 가격 최적화
"광고 비용 X원일 때 ROI가 양수인 최대 X는?"
→ 비용을 이진 탐색

### 5. 배열/리스트의 삽입 위치 결정
`Array.prototype.sort()` 후 이진 탐색으로 삽입 위치를 찾아 O(log n)에 검색.
