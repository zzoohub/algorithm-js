# FFT (Fast Fourier Transform, 고속 푸리에 변환)

## 왜 필요한가?

스마트폰으로 음악을 들을 때, 사진을 JPEG으로 저장할 때, Wi-Fi로 인터넷을 쓸 때 — **모두 FFT가 돌아가고 있다.** FFT 없이는 현대 디지털 기술이 불가능하다.

```
100만 개 항의 두 다항식을 곱하려면?
  나이브 방법: 1,000,000 × 1,000,000 = 1조 번의 곱셈 → 수 시간
  FFT:        20,000,000번의 연산                     → 0.1초
  → 5만 배 빠름!
```

## 개념

**O(n²)의 다항식 곱셈을 O(n log n)으로** 줄이는 알고리즘.
"20세기에 발견된 가장 중요한 수치 알고리즘" 중 하나로 불린다.

```
다항식 곱셈:
  A(x) = 1 + 2x + 3x²
  B(x) = 4 + 5x

나이브: 각 항을 일일이 곱함 → O(n²)
  = 4 + 5x + 8x + 10x² + 12x² + 15x³
  = 4 + 13x + 22x² + 15x³

FFT: O(n log n)으로 같은 결과를 얻음
```

## 핵심 아이디어: 표현의 전환

다항식을 표현하는 두 가지 방법:

```
계수 표현:  A(x) = [1, 2, 3]  (계수 나열)
점-값 표현: A(x₀)=6, A(x₁)=2, A(x₂)=... (여러 점에서의 값)
```

| 연산 | 계수 표현 | 점-값 표현 |
|------|-----------|-----------|
| 곱셈 | O(n²) | **O(n)** (점별 곱) |
| 변환 비용 | — | O(n²) (나이브), **O(n log n)** (FFT) |

**전략**: 계수 → (FFT) → 점-값에서 곱셈 → (IFFT) → 계수

```
A = [1,2,3]    B = [4,5]
    ↓ FFT          ↓ FFT         O(n log n)
Â = [6,...]    B̂ = [9,...]
    ↓ 점별 곱                     O(n)
Ĉ = [54,...]
    ↓ IFFT                       O(n log n)
C = [4,13,22,15]                 총 O(n log n)!
```

## 단위근 (Roots of Unity)

"어떤 점들에서 평가할 것인가?" — 아무 점이나 되는 게 아니라, **복소수의 단위근**에서 평가해야 분할정복이 가능하다.

### n번째 단위근

```
wₙ = e^(2πi/n) = cos(2π/n) + i·sin(2π/n)

wₙ의 거듭제곱들은 복소 평면의 단위원 위에 균등 배치:

        w₈⁰ = 1
   w₈⁷  |  w₈¹
        \|/
w₈⁶ ────●──── w₈² ← 단위원
        /|\
   w₈⁵  |  w₈³
        w₈⁴ = -1

핵심 성질: wₙ^(n/2) = -1 (반바퀴 돌면 부호 반전)
```

이 성질 덕분에 **Butterfly 연산**이 가능:
```
A(wₙ^k)       = A_even(wₙ^(2k)) + wₙ^k · A_odd(wₙ^(2k))
A(wₙ^(k+n/2)) = A_even(wₙ^(2k)) - wₙ^k · A_odd(wₙ^(2k))
                                    ↑ 부호만 반전!
→ 한 번에 두 값을 계산
```

## 분할정복 (Cooley-Tukey)

```
A(x) = a₀ + a₁x + a₂x² + a₃x³ + a₄x⁴ + a₅x⁵

짝수 차수: A_even(y) = a₀ + a₂y + a₄y²    (y = x²)
홀수 차수: A_odd(y)  = a₁ + a₃y + a₅y²

A(x) = A_even(x²) + x · A_odd(x²)

크기 n → 크기 n/2 두 개로 분할 → T(n) = 2T(n/2) + O(n) = O(n log n)
```

### Butterfly 다이어그램

```
입력:                      출력:
a₀ ──┐        ┌── A(w⁰)
a₂ ──┼─ even ─┤
a₄ ──┘   FFT  └── ...
                  ↗ ↘ (butterfly)
a₁ ──┐        ┌── ...
a₃ ──┼─ odd  ─┤
a₅ ──┘   FFT  └── A(w⁵)
```

## IFFT (역변환)

놀랍게도 **FFT와 거의 같은 코드**로 역변환을 수행할 수 있다.
차이점 두 가지:
1. 단위근의 방향을 반대로 (e^(-2πi/n))
2. 결과를 n으로 나눔

## 큰 수 곱셈

FFT의 실용적 응용: 각 자릿수를 계수로 보면 큰 수 곱셈 = 다항식 곱셈.

```
123 × 456:
  [3, 2, 1] × [6, 5, 4]    (역순 = 낮은 자릿수부터)
  = [18, 27, 28, 13, 4]     (다항식 곱셈)
  → carry 처리 → 56088

기존: O(n²) 필기 곱셈
FFT:  O(n log n)
```

Python의 큰 수 곱셈, GMP 라이브러리가 내부적으로 FFT 변형(NTT)을 사용.

## 시간 복잡도

| | 나이브 | FFT |
|---|---|---|
| 다항식 곱셈 | O(n²) | **O(n log n)** |
| DFT/IDFT | O(n²) | **O(n log n)** |
| 공간 | O(n) | O(n) |

## 실무에서의 활용

### 1. 오디오/신호 처리
음악 스트리밍(Spotify, Apple Music)의 이퀄라이저, 노이즈 캔슬링.
시간 영역의 파형을 주파수 영역으로 변환하여 특정 주파수를 증폭/감쇠.
모든 오디오 코덱(MP3, AAC)의 핵심 단계.

### 2. 이미지 압축 (JPEG)
JPEG 압축은 DCT(Discrete Cosine Transform)를 사용하는데, 이것은 FFT의 변형.
이미지를 주파수 성분으로 분해하여 고주파(디테일)를 버리는 것이 압축의 원리.

### 3. 통신 (OFDM, 5G)
Wi-Fi, 4G/5G, DVB-T(디지털 TV)가 OFDM(직교 주파수 분할 다중화)을 사용.
OFDM의 핵심 연산이 FFT/IFFT. 스마트폰의 모뎀 칩이 초당 수십억 번의 FFT를 수행.

### 4. 과학 계산
분자 동역학 시뮬레이션, 기상 예측, 천문학 데이터 분석.
편미분방정식의 수치 해법에서 FFT로 주파수 영역 계산.

### 5. 문자열 매칭 (와일드카드)
와일드카드가 포함된 패턴 매칭을 다항식 곱셈으로 변환.
컨볼루션(convolution) = 다항식 곱셈 = FFT로 O(n log n).

### 왜 알아야 하는가
FFT는 **"문제를 다른 도메인(주파수 영역)으로 변환하면 더 쉬워진다"**는 사고의 결정체이다. 이 "변환 후 처리" 패턴은 로그 스케일, 라플라스 변환, 임베딩 등 수학과 CS 전반에서 반복되는 가장 강력한 문제 해결 전략 중 하나다.
