# Shortest Path (최단 경로 알고리즘)

## Dijkstra 알고리즘

### 개념
**음이 아닌 가중치** 그래프에서 한 정점에서 다른 모든 정점까지의 최단 경로를 구한다.
**탐욕(Greedy)** 알고리즘 — 매 단계 현재까지 가장 가까운 노드를 선택.

### 동작 원리
```
그래프:        A --4-- B
              |       |
              2       1
              |       |
              C --3-- D

시작: A
1. A(0) 선택 → B=4, C=2 업데이트
2. C(2) 선택 → D=2+3=5 업데이트
3. B(4) 선택 → D=min(5, 4+1)=5 (변경 없음)
4. D(5) 선택

최단 거리: A=0, B=4, C=2, D=5
```

### 핵심: Relaxation (완화)
```
if (dist[current] + weight < dist[neighbor]) {
  dist[neighbor] = dist[current] + weight;
}
```
"A를 거쳐 B로 가는 게 지금까지 알려진 B까지의 경로보다 짧은가?"

### 시간 복잡도
| 구현 | 시간 |
|------|------|
| 배열 (이 구현) | O(V²) |
| Min Heap | O((V+E) log V) |
| 피보나치 힙 | O(V log V + E) |

### 제한: 음수 가중치 불가
음수 간선이 있으면 "이미 처리 완료"한 노드가 나중에 더 짧아질 수 있다.
→ Bellman-Ford를 사용.

---

## Bellman-Ford 알고리즘

### 개념
**음수 가중치**도 처리 가능. 음수 사이클도 감지할 수 있다.
모든 간선을 V-1번 반복해서 완화한다.

### 동작 원리
- V개의 정점이 있으면, 최단 경로는 최대 V-1개의 간선을 거친다
- V-1번 모든 간선을 완화하면 최단 거리가 확정된다
- V번째에도 완화가 가능하면 → **음수 사이클 존재**

### 시간 복잡도
- O(V × E) — Dijkstra보다 느리지만, 음수 가중치를 처리

---

## Dijkstra vs Bellman-Ford

| | Dijkstra | Bellman-Ford |
|---|---|---|
| 음수 가중치 | X | O |
| 음수 사이클 감지 | X | O |
| 시간 (Min Heap) | O((V+E) log V) | O(V × E) |
| 구현 난이도 | 중간 | 쉬움 |

## 실무에서의 활용

### 1. 네비게이션 (Google Maps, Naver Map)
교차로 = 정점, 도로 = 간선, 이동 시간 = 가중치.
실제로는 A* (Dijkstra + 휴리스틱)를 사용해서 더 빠르게 탐색.

### 2. 네트워크 라우팅
OSPF (Open Shortest Path First) 프로토콜이 Dijkstra를 사용.
라우터들이 네트워크 토폴로지를 공유하고 최적 경로를 계산.

### 3. 게임 AI 경로 탐색
NPC가 맵에서 목적지까지 이동하는 경로를 A*/Dijkstra로 계산.

### 4. 환율 차익 거래 (음수 사이클)
환율의 로그를 가중치로 사용하면, 음수 사이클 = 차익 거래 기회.
Bellman-Ford로 이런 기회를 탐지.
