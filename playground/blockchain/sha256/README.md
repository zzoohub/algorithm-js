# SHA-256 (Secure Hash Algorithm)

## 개념

임의 길이의 입력을 **고정 256비트(64자 hex)**로 변환하는 암호학적 해시 함수.
블록체인의 가장 기본적인 빌딩 블록.

```
sha256("hello")      → 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
sha256("hello!")     → ce06092fb948d9ffac7d1a376e404b26b7575bcc11ee05a4615fef4fec3a308b
sha256("hello!!")    → 6e53353beda78ea5e591ffc86a32790da04e03ae5fd40e1ebe7dbe5cfce4c710

→ 1글자만 달라져도 완전히 다른 해시 (눈사태 효과)
```

## 5가지 핵심 성질

### 1. 결정적 (Deterministic)
같은 입력 → 항상 같은 출력. 전 세계 어디서 누가 계산해도 동일.

### 2. 빠른 계산 (Fast)
수 GB의 파일도 밀리초 안에 해시 가능.

### 3. 역상 저항성 (Pre-image Resistance)
```
해시 → 원본?  불가능!

"2cf24dba..." → "hello"?  ← 알 수 없음
유일한 방법: 모든 가능한 입력을 시도 (brute force)
256비트 공간 = 2^256 ≈ 10^77 가지 → 우주의 원자 수보다 많음
```

### 4. 충돌 저항성 (Collision Resistance)
```
sha256(A) = sha256(B) 인 서로 다른 A, B를 찾는 것이 사실상 불가능.
Birthday Attack으로도 2^128 번의 시도 필요.
```

### 5. 눈사태 효과 (Avalanche Effect)
```
입력 1비트만 바뀌어도 출력의 약 50% 비트가 변함.
→ "비슷한 입력"으로 "비슷한 해시"를 만들 수 없음
→ 해시를 보고 원본에 대한 어떤 정보도 유추 불가
```

## 내부 동작

### 전처리 (Padding)
```
메시지: "abc" (3바이트 = 24비트)
→ 패딩: [61 62 63 80 00...00 00 00 00 18]  (512비트로 맞춤)
                  ↑              ↑
              1비트 추가    원본 길이(24) big-endian
```

### 압축 함수 (64라운드)
```
512비트 블록마다 64라운드의 비트 연산:

각 라운드:
  T₁ = h + Σ₁(e) + Ch(e,f,g) + Kᵢ + Wᵢ
  T₂ = Σ₀(a) + Maj(a,b,c)
  
  h=g, g=f, f=e, e=d+T₁, d=c, c=b, b=a, a=T₁+T₂

Ch(x,y,z) = (x AND y) XOR (NOT x AND z)     "선택"
Maj(x,y,z) = (x AND y) XOR (x AND z) XOR (y AND z)  "다수결"
Σ₀, Σ₁ = 비트 회전과 XOR의 조합

이 과정이 해시의 "뒤섞기"를 담당 → 눈사태 효과 생성
```

## 블록체인에서의 사용

### 1. 블록 해시 (Block Hash)
```
블록 헤더 = {이전 블록 해시, 머클루트, 타임스탬프, 난이도, nonce}
블록 해시 = SHA256(SHA256(블록 헤더))

→ 블록의 "지문". 1비트라도 바뀌면 해시가 완전히 달라짐
→ 블록체인의 "변조 방지"의 원리
```

### 2. 채굴 (Proof of Work)
```
"SHA256(블록 헤더 + nonce) < 목표값" 인 nonce를 찾아라

nonce=0: SHA256(...) = 8a3f...  ✗ (너무 큼)
nonce=1: SHA256(...) = f21c...  ✗
nonce=2: SHA256(...) = 0000...  ✓ 앞에 0이 충분히 많음!

→ 역상 저항성 때문에 시행착오 외에는 방법이 없음
→ 이것이 "작업 증명"의 "작업"
```

### 3. 주소 생성
```
비트코인 주소 = Base58(RIPEMD160(SHA256(공개키)))
→ 공개키에서 주소는 계산 가능
→ 주소에서 공개키를 역추적하는 건 불가능
```

### 4. 머클 트리
```
거래들의 해시를 쌍으로 묶어 올라가며 하나의 루트 해시를 만듦.
→ 어떤 거래가 블록에 포함되었는지를 효율적으로 증명
```

## SHA-256 vs 다른 해시

| | MD5 | SHA-1 | **SHA-256** | SHA-3 |
|---|---|---|---|---|
| 출력 크기 | 128비트 | 160비트 | **256비트** | 가변 |
| 안전성 | 깨짐 | 깨짐 | **안전** | 안전 |
| 속도 | 빠름 | 빠름 | 보통 | 보통 |
| 용도 | 체크섬만 | 비추천 | **블록체인, TLS** | 차세대 |

## 왜 알아야 하는가

SHA-256을 이해하면:
- 블록체인의 보안 모델 전체를 이해할 수 있다
- "해시가 안전하다"가 정확히 무슨 의미인지 알 수 있다
- PoW 채굴이 왜 에너지를 소모하는지 이해할 수 있다
- 디지털 서명, 인증서, HTTPS의 기초를 이해할 수 있다
