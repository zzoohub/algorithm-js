# Proof of Work (작업 증명)

## 개념

**"건초 더미에서 바늘 찾기, 하지만 검증은 한 순간."**

작업 증명은 어떤 계산 작업을 수행했다는 것을 증명하는 메커니즘이다.
블록체인에서는 새 블록을 추가할 자격을 얻기 위해 채굴자가 풀어야 하는 퍼즐이다.

```
문제: hash(데이터 + nonce)의 앞부분이 0000...으로 시작하는 nonce를 찾아라

nonce=0:  hash("블록데이터:0")  = a3f2c8...  ✗
nonce=1:  hash("블록데이터:1")  = 7b1d09...  ✗
nonce=2:  hash("블록데이터:2")  = f8e412...  ✗
  ...수천~수백만 번 반복...
nonce=84371: hash("블록데이터:84371") = 0000ab...  ✓ 발견!

검증: hash("블록데이터:84371") 한 번만 계산 → 0000으로 시작? → 유효!
```

## 왜 작동하는가: 해시의 역상 저항성

SHA-256의 **역상 저항성(pre-image resistance)** 때문에 brute force가 유일한 방법이다.

```
순방향: hash(입력) → 출력          ← O(1), 즉시 계산
역방향: 출력 → hash(???) = 출력    ← 불가능! 시행착오 외 방법 없음

해시 함수는 "일방통행 함수":
  → 입력에서 출력으로 가는 건 고속도로
  → 출력에서 입력으로 돌아가는 건 벽
```

## 난이도와 채굴 시간

### 선행 0의 의미

해시의 각 자리는 hex(0~f, 16가지). 한 자리가 `0`일 확률 = 1/16.

```
난이도 1: 0xxxxxxx  → 확률 1/16     → 평균 ~16회 시도
난이도 2: 00xxxxxx  → 확률 1/256    → 평균 ~256회 시도
난이도 3: 000xxxxx  → 확률 1/4,096  → 평균 ~4,096회 시도
난이도 4: 0000xxxx  → 확률 1/65,536 → 평균 ~65,536회 시도

→ 난이도가 1 증가할 때마다 기대 시도 횟수가 16배 증가!
→ 실제 비트코인(SHA-256, 비트 단위)에서는 1비트당 2배 증가
```

### 기하급수적 증가 시각화

```
난이도 1: ████  (~16회)
난이도 2: ████████████████████████████  (~256회)
난이도 3: ████████████████████████████████████████████████████...  (~4,096회)

→ 선형이 아닌 지수적 증가. 이것이 PoW에 엄청난 에너지가 드는 이유.
```

## 비트코인의 난이도 조정

### 메커니즘

비트코인은 **2016블록마다 (약 2주)** 난이도를 조정하여 **블록당 ~10분**을 유지한다.

```
목표: 2016블록 × 10분 = 20,160분 = 2주

실제 소요 시간이 2주보다 짧으면 → 너무 쉬웠다 → 난이도 ↑
실제 소요 시간이 2주보다 길면   → 너무 어려웠다 → 난이도 ↓

공식:
새 난이도 = 현재 난이도 × (목표 시간 ÷ 실제 시간)

안전장치: 한 번에 최대 4배까지만 변경 가능 (급격한 변동 방지)
```

### 왜 10분인가

```
너무 짧으면 (예: 1초):
  - 블록이 네트워크에 전파되기 전에 다음 블록이 생김
  - 포크(fork)가 빈번 → 합의 불안정

너무 길면 (예: 1시간):
  - 거래 확인이 느려짐
  - 사용자 경험 저하

10분 = 전파 시간과 확인 속도의 균형점
```

## 에너지 논쟁

### 왜 PoW는 에너지를 소비하는가

```
채굴 = 해시를 반복 계산하는 것 = CPU/GPU를 최대로 가동하는 것

비트코인 네트워크 전체:
  - 초당 ~600 EH/s (600 × 10^18 해시/초) — 2024년 기준
  - 연간 전력 소비: ~150 TWh (아르헨티나 한 나라 수준)
  - 이 에너지의 대부분은 "틀린 nonce"를 계산하는 데 사용됨

그러나 이 에너지 소비가 곧 보안:
  - 공격자가 네트워크를 장악하려면 같은 수준의 에너지를 투입해야 함
  - 경제적으로 공격보다 정직한 채굴이 더 이득이 되도록 설계됨
```

### 대안

| 합의 메커니즘 | 에너지 | 보안 모델 |
|---|---|---|
| **PoW** | 매우 높음 | 계산 능력 (해시파워) |
| **PoS** | 매우 낮음 | 담보금 (스테이킹) |
| **DPoS** | 낮음 | 위임된 검증자 투표 |
| **PoA** | 최소 | 신원 기반 권한 |

## 51% 공격

### 원리

블록체인에서 "가장 긴 체인이 유효한 체인"이다.
만약 누군가 전체 해시파워의 51% 이상을 가지면:

```
정상 체인:  A → B → C → D → E
                    ↘
공격 체인:            C' → D' → E' → F'  (더 긴 체인)

공격자는:
1. 자신의 거래가 포함된 블록 C를 만든다 (물건 구매)
2. 비밀리에 C'부터 다른 체인을 채굴한다 (거래 없는 버전)
3. 정상 체인보다 길어지면 공개한다
4. 네트워크는 더 긴 체인을 채택 → 원래 거래가 사라짐 (이중 지불!)
```

### 왜 어려운가

```
비트코인 해시레이트: ~600 EH/s
51%를 확보하려면: ~306 EH/s

필요한 장비: 최신 ASIC 채굴기 수백만 대
예상 비용: 수십억 달러 (장비) + 막대한 전력비
공격 성공해도: 비트코인 가격 폭락 → 공격자의 장비 가치도 폭락

→ 경제적으로 비합리적 = 게임 이론적 보안
```

## PoW vs PoS 비교

| 항목 | PoW (작업 증명) | PoS (지분 증명) |
|---|---|---|
| **검증 방식** | 해시 퍼즐 풀기 | 스테이킹된 자산 기반 |
| **에너지 소비** | 매우 높음 | 매우 낮음 (~99.95% 절감) |
| **진입 장벽** | 채굴 장비 (ASIC, GPU) | 암호화폐 보유량 |
| **보안 근거** | 물리적 자원 (전력, 하드웨어) | 경제적 담보 (슬래싱 패널티) |
| **공격 비용** | 51% 해시파워 확보 | 전체 스테이킹의 33% 확보 |
| **탈중앙화** | 채굴 풀 집중화 우려 | 부의 집중화 우려 |
| **블록 생성** | 확률적 (운에 의존) | 결정적 또는 준결정적 |
| **최종성** | 확률적 (6블록 ≈ 1시간) | 보통 더 빠른 최종성 |
| **포크 처리** | 가장 긴 체인 = 유효 | 슬래싱으로 이중 서명 처벌 |
| **하드웨어 낭비** | ASIC은 채굴 외 용도 없음 | 별도 하드웨어 불필요 |

## 실제 사용 사례

### PoW를 사용하는 블록체인

- **비트코인 (Bitcoin)**: SHA-256 기반, 가장 오래되고 안전한 PoW 네트워크
- **라이트코인 (Litecoin)**: Scrypt 기반, 비트코인보다 빠른 블록 시간 (2.5분)
- **도지코인 (Dogecoin)**: Scrypt 기반, 라이트코인과 병합 채굴
- **모네로 (Monero)**: RandomX 기반, ASIC 저항성 (CPU 채굴 유지)

### 이더리움: PoW → PoS 전환 (The Merge)

```
2022년 9월 15일 "The Merge":
  이더리움이 PoW에서 PoS로 전환

결과:
  - 에너지 소비 ~99.95% 감소
  - 초당 해시 계산 → 검증자의 스테이킹으로 대체
  - GPU 채굴자들은 다른 코인으로 이동하거나 철수

의미:
  - PoW 없이도 블록체인 보안이 가능함을 대규모로 증명
  - 그러나 비트코인 커뮤니티는 PoW의 물리적 보안을 더 신뢰
```

## 왜 알아야 하는가

Proof of Work를 이해하면:
- 블록체인이 "신뢰 없이 합의"하는 원리를 이해할 수 있다
- 비트코인 채굴이 왜 에너지를 소모하는지 정확히 알 수 있다
- 51% 공격의 위협과 그 한계를 판단할 수 있다
- PoW vs PoS 논쟁에서 근거 있는 의견을 가질 수 있다
- 난이도 조정이 네트워크 안정성에 어떻게 기여하는지 알 수 있다
