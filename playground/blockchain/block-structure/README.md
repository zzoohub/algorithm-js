# 블록 구조 (Block Structure)

## 블록이란?

블록체인의 기본 단위. **헤더**(메타데이터)와 **바디**(트랜잭션 목록)로 구성된다.
은행의 장부 한 "페이지"라고 생각하면 된다 — 거래 내역이 기록되고, 페이지 번호와 이전 페이지 참조가 있다.

```
┌─────────────────────────────────────┐
│            블록 (Block)              │
├─────────────────────────────────────┤
│  ┌───────────────────────────────┐  │
│  │       헤더 (Header)           │  │
│  ├───────────────────────────────┤  │
│  │  index         : 블록 번호    │  │
│  │  timestamp     : 생성 시각    │  │
│  │  previousHash  : 이전 블록 해시│  │
│  │  nonce         : 채굴 값      │  │
│  │  hash          : 이 블록 해시  │  │
│  └───────────────────────────────┘  │
│  ┌───────────────────────────────┐  │
│  │       바디 (Body)             │  │
│  ├───────────────────────────────┤  │
│  │  transactions:                │  │
│  │    - "Alice → Bob: 10 BTC"    │  │
│  │    - "Charlie → Dave: 5 BTC"  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

## 블록 헤더 필드 설명

| 필드 | 설명 | 비트코인에서 |
|---|---|---|
| **index** | 블록 번호 (0부터 시작) | Block Height |
| **timestamp** | 블록이 만들어진 시각 | Unix timestamp (4바이트) |
| **previousHash** | 이전 블록의 해시 | 32바이트 SHA-256 해시 |
| **nonce** | PoW에서 채굴자가 조정하는 값 | 4바이트 정수 |
| **hash** | 이 블록의 해시 (헤더 전체를 해시) | SHA256(SHA256(header)) |
| transactions | 블록에 포함된 거래 목록 | Merkle Root로 요약 |

비트코인 블록 헤더에는 추가로 `version`, `merkleRoot`, `difficulty(bits)` 필드가 있다.

## 체인 연결: 해시로 만든 신뢰의 사슬

각 블록은 이전 블록의 해시를 `previousHash`에 저장한다.
이것이 블록들을 **체인(chain)**으로 연결하는 핵심 메커니즘이다.

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Block 0    │    │   Block 1    │    │   Block 2    │
│  (Genesis)   │    │              │    │              │
├──────────────┤    ├──────────────┤    ├──────────────┤
│ prevHash: "0"│    │ prevHash:    │    │ prevHash:    │
│              │◄───│  hash₀       │◄───│  hash₁       │
│ hash: hash₀  │    │ hash: hash₁  │    │ hash: hash₂  │
│              │    │              │    │              │
│ tx: Genesis  │    │ tx: A→B 10  │    │ tx: B→C 5   │
│     Block    │    │     C→D  5  │    │     D→A 7   │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │
       └─── 화살표 방향 = "이 해시를 참조한다" ───┘
```

**핵심**: Block 1의 `previousHash`는 Block 0의 `hash`와 정확히 같다.
만약 Block 0의 내용이 바뀌면 `hash₀`가 달라지고, Block 1의 `previousHash`와 불일치 → 체인 끊김!

## 왜 변조가 감지되는가?

### 시나리오: Block 1의 거래 내역을 위조

```
원본:  "Alice → Bob: 10 BTC"
위조:  "Alice → Mallory: 100 BTC"

1. Block 1의 transactions가 바뀜
   → Block 1의 해시가 달라짐 (hash₁ ≠ hash₁')
   
2. Block 1에 저장된 hash는 옛날 값(hash₁) 그대로
   → calculateHash() 결과(hash₁')와 불일치
   → 검증 실패! "블록 내용이 변조되었습니다"

3. 설령 hash도 다시 계산해서 업데이트한다면?
   → Block 2의 previousHash(= hash₁)와 Block 1의 새 hash(= hash₁')가 불일치
   → 검증 실패! "체인 연결이 끊어졌습니다"

4. Block 2도 수정한다면? Block 3도? ... 끝까지?
   → PoW 때문에 각 블록마다 재채굴 필요
   → 네트워크의 나머지 노드들이 이미 정직한 체인으로 진행 중
   → 공격자의 체인이 정직한 체인보다 빠를 수 없음 → 공격 실패
```

### 변조 감지의 두 가지 검증

```
검증 1: 블록 내부 일관성
  block.hash === block.calculateHash()?
  → NO라면: 블록 데이터가 변조됨

검증 2: 체인 연결 일관성
  block[i].previousHash === block[i-1].hash?
  → NO라면: 체인이 끊어짐 (중간에 블록이 바뀌었거나 삽입됨)
```

## 제네시스 블록 (Genesis Block)

체인의 첫 번째 블록. 이전 블록이 없으므로 `previousHash = "0"` (또는 임의의 값).

- 블록체인의 "기원"
- 모든 노드가 동일한 제네시스 블록으로 시작해야 함 (하드코딩)
- 비트코인 제네시스 블록: 2009-01-03, 사토시 나카모토 생성
  - 코인베이스 메시지: `"The Times 03/Jan/2009 Chancellor on brink of second bailout for banks"`
  - 이 신문 헤드라인을 넣어 특정 날짜 이후에 만들어졌음을 증명

## 실제 비트코인 블록 구조

비트코인의 실제 블록은 더 복잡하지만, 원리는 동일하다:

```
비트코인 블록 헤더 (80바이트):
┌────────────────────────────────┐
│ version        (4바이트)        │  프로토콜 버전
│ previousHash   (32바이트)       │  이전 블록의 SHA-256 해시
│ merkleRoot     (32바이트)       │  모든 트랜잭션의 머클 루트
│ timestamp      (4바이트)        │  블록 생성 시각
│ bits           (4바이트)        │  난이도 목표 (압축 형태)
│ nonce          (4바이트)        │  채굴 시 조정하는 값
└────────────────────────────────┘
```

**블록 탐색기(Block Explorer) 개념**:
- blockchain.com, blockchair.com 등에서 실제 블록을 조회 가능
- 블록 높이(height), 해시, 포함된 트랜잭션 수, 채굴자, 보상 등 확인
- 트랜잭션 ID로 특정 거래의 확인(confirmation) 수, 수수료 등 조회
- 블록 해시를 클릭하면 헤더의 모든 필드를 볼 수 있음

## 다른 주제와의 연결

### SHA-256 (해싱)
블록의 해시를 계산하는 데 SHA-256 사용.
`hash = SHA256(SHA256(blockHeader))` — 비트코인은 이중 해싱.
이 구현에서는 학습용으로 FNV-1a를 사용했지만, 원리는 동일하다.

### 머클 트리 (Merkle Tree)
블록 바디의 트랜잭션들을 하나의 해시(`merkleRoot`)로 요약.
수천 개의 트랜잭션도 32바이트 하나로 표현 가능.
특정 트랜잭션이 블록에 포함되었는지 `O(log n)`으로 증명 (SPV).

### 작업 증명 (Proof of Work)
"해시의 앞 N자리가 0"인 nonce를 찾는 과정.
이것이 블록 추가에 비용을 부과 → 변조 시 재채굴 비용이 막대.
난이도가 1 올라갈 때마다 필요한 시도 횟수가 약 16배 증가 (hex 기준).

### ECDSA (디지털 서명)
트랜잭션의 "보낸 사람"이 실제 소유자인지 검증.
블록에 포함된 각 트랜잭션은 보낸 사람의 개인키로 서명되어 있다.
블록 구조는 "무엇이 기록되었는가"를, ECDSA는 "누가 승인했는가"를 담당.
