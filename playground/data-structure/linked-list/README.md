# Linked List (연결 리스트)

## 개념

각 노드가 **데이터**와 **다음 노드의 참조(포인터)**를 가진 선형 자료구조.
배열과 달리 메모리에 연속적으로 저장되지 않는다.

```
[data|next] → [data|next] → [data|next] → null
  head                         tail
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| 앞에 삽입 (prepend) | O(1) |
| 뒤에 삽입 (append) | O(1) (tail 포인터 유지 시) |
| 삭제 (특정 값) | O(n) |
| 검색 | O(n) |
| 인덱스 접근 | O(n) — 배열은 O(1) |

## 배열 vs 연결 리스트

| | 배열 | 연결 리스트 |
|---|---|---|
| 인덱스 접근 | O(1) | O(n) |
| 앞에 삽입/삭제 | O(n) | O(1) |
| 메모리 | 연속, 캐시 친화적 | 분산, 포인터 오버헤드 |
| 크기 변경 | 재할당 필요 | 동적 |

## 동작 원리

- **append**: 새 노드 생성 → tail.next에 연결 → tail 갱신
- **prepend**: 새 노드 생성 → 새 노드.next = head → head 갱신
- **delete**: 이전 노드의 next를 삭제 대상의 next로 연결 (건너뛰기)

## 실무에서의 활용

### 1. React Fiber 아키텍처
React의 내부 렌더링 엔진(Fiber)은 연결 리스트 구조로 컴포넌트 트리를 관리한다.
각 Fiber 노드가 child, sibling, return 포인터를 가진다.

### 2. 브라우저 히스토리
이전/다음 페이지 이동이 이중 연결 리스트와 유사하게 동작한다.

### 3. OS 메모리 관리
운영체제의 프리 리스트(free list)가 사용 가능한 메모리 블록을 연결 리스트로 관리한다.

### 4. LRU Cache의 기반
HashMap + Doubly Linked List 조합으로 O(1) 캐시를 만든다.

### 5. 음악 플레이어 재생목록
이전곡/다음곡 이동이 이중 연결 리스트와 동일한 구조.
