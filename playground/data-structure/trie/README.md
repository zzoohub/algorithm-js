# Trie (트라이)

## 개념

문자열을 **문자 단위로 트리에 저장**하는 자료구조.
공통 접두사를 공유해서 메모리를 절약하고, 접두사 검색이 매우 빠르다.

```
insert: "cat", "car", "card", "dog"

        root
       /    \
      c      d
      |      |
      a      o
     / \     |
    t   r    g*
    *   |
        d*

(* = 단어의 끝)
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| 삽입 | O(m) — m은 문자열 길이 |
| 검색 | O(m) |
| 접두사 검색 | O(m) |
| 자동완성 | O(m + k) — k는 결과 수 |

해시 테이블보다 접두사 기반 검색에서 압도적으로 유리하다.

## 동작 원리

### 삽입
문자열의 각 문자를 따라 트리를 내려가며 노드를 생성한다.
마지막 노드에 isEnd = true 표시.

### 검색
문자를 따라 내려가다가 중간에 경로가 없으면 false.
끝까지 도달했는데 isEnd가 false면 해당 문자열은 저장되지 않은 것.

### 삭제
재귀적으로 내려가서 isEnd를 해제하고, 자식이 없는 노드는 거슬러 올라가며 정리.

## 실무에서의 활용

### 1. 검색 엔진 자동완성
"app"을 입력하면 "apple", "application", "approach" 등을 즉시 제안.
Google 검색창, IDE의 코드 자동완성이 Trie 변형을 사용한다.

### 2. 라우터 매칭
Express/Hono 같은 웹 프레임워크가 URL 패턴을 Trie로 매칭한다.
`/api/users/:id` 같은 경로를 효율적으로 탐색.

### 3. IP 라우팅 (Longest Prefix Match)
네트워크 라우터가 IP 주소의 가장 긴 접두사를 매칭할 때 사용.

### 4. 스펠 체커
입력한 단어가 사전에 있는지 빠르게 확인하고, 비슷한 단어를 추천.

### 5. T9 키패드 예측
피처폰에서 숫자 조합으로 단어를 예측하던 기능이 Trie 기반.
