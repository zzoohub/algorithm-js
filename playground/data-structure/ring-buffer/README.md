# Ring Buffer (원형 버퍼, Circular Buffer)

## 개념

**고정 크기 배열**에서 head와 tail 포인터가 원형으로 순환하는 FIFO 자료구조.
가득 차면 가장 오래된 데이터를 덮어쓴다.

```
용량 4:
write(A) write(B) write(C) write(D) → [A, B, C, D] (가득 참)
write(E) → [E, B, C, D]  ← A를 덮어씀, head가 B로 이동
                head=1, tail=1

원형 인덱스: tail = (tail + 1) % capacity
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| write | O(1) |
| read | O(1) |
| peek | O(1) |

메모리 할당/해제가 전혀 없다 — 초기에 한 번만 할당.

## 동작 원리

### 모듈러 연산으로 순환
배열의 끝에 도달하면 다시 처음으로 돌아간다.
```
다음 위치 = (현재 위치 + 1) % 용량
```

### head와 tail
- **tail**: 다음 쓸 위치. write하면 전진.
- **head**: 가장 오래된 데이터 위치. read하면 전진.
- 가득 찼을 때 write하면 head도 함께 전진 (오래된 데이터 버림).

### 왜 고정 크기인가?
동적 배열(push/pop)과 달리 메모리 재할당이 없어서:
- 할당/해제 오버헤드 없음
- 메모리 단편화 없음
- 예측 가능한 지연 시간 (실시간 시스템에 중요)

## 실무에서의 활용

### 1. OS 커널 버퍼
키보드 입력 버퍼, 시리얼 포트 버퍼가 링 버퍼이다.
키를 빠르게 여러 번 누르면 버퍼에 쌓이고, OS가 순서대로 읽는다.

### 2. 네트워크 소켓 버퍼
TCP 수신 버퍼가 링 버퍼이다. 데이터가 도착하면 tail에 쓰고,
애플리케이션이 read()하면 head에서 읽는다.

### 3. 오디오/비디오 스트리밍
오디오 장치가 일정 속도로 샘플을 소비하는 동안
프로듀서가 링 버퍼에 쓴다. 버퍼 언더런 = 소리가 끊김.

### 4. 로그 시스템
최근 N개의 로그만 메모리에 유지할 때 링 버퍼가 적합하다.
Linux의 `dmesg`(커널 로그)가 링 버퍼이다.

### 5. Producer-Consumer 패턴
멀티스레드에서 락 없는(lock-free) 큐를 구현할 때 링 버퍼를 사용한다.
LMAX Disruptor(고성능 금융 시스템)가 대표적인 예.
