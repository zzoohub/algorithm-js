# Fenwick Tree (펜윅 트리 / Binary Indexed Tree)

## 개념

**Segment Tree의 경량 버전.** 구간 합(prefix sum)과 점 갱신을 O(log n)에 처리한다.
Segment Tree가 만능 도구라면, Fenwick Tree는 **작고 날카로운 칼** — 구간 합에 특화되어 코드가 20줄이고, 상수가 작아서 실제로 더 빠르다.

```
원본 배열: [3, 2, -1, 6, 5, 4, -3, 3, 7, 2]

Fenwick 인덱스(1-based)가 담당하는 구간:
  1  (0001) → [3]          = 3
  2  (0010) → [3, 2]       = 5
  3  (0011) → [-1]         = -1
  4  (0100) → [3, 2, -1, 6] = 10
  5  (0101) → [5]          = 5
  6  (0110) → [5, 4]       = 9
  ...
```

## 핵심 트릭: `i & (-i)`

2의 보수에서 `-i`는 비트를 반전한 뒤 +1. 그래서 `i & (-i)`는 **가장 낮은 1 비트**를 추출한다.

```
i = 12 = 1100₂
-i     = 0100₂  (반전: 0011 → +1: 0100)
i & -i = 0100₂ = 4

→ 인덱스 12는 4개의 원소를 담당 (arr[9..12])
```

이 비트 트릭으로:
- **Prefix Sum (아래로)**: `i -= (i & -i)` — 구간을 쪼개며 합산
- **Update (위로)**: `i += (i & -i)` — 영향받는 구간을 올라가며 갱신

### Prefix Sum 예시
```
prefixSum(7): 인덱스 1~7의 합

7 = 0111₂  → tree[7] (arr[7]만의 값)
  -= 0001₂
6 = 0110₂  → tree[6] (arr[5..6]의 합)
  -= 0010₂
4 = 0100₂  → tree[4] (arr[1..4]의 합)
  -= 0100₂
0 → 끝

총 3번의 덧셈으로 7개 원소의 합을 구함!
```

## Segment Tree와의 비교

| | Fenwick Tree | Segment Tree |
|---|---|---|
| 구간 합 질의 | O(log n) | O(log n) |
| 점 갱신 | O(log n) | O(log n) |
| 구간 갱신 | O(log n)* | O(log n) + Lazy |
| 구간 최솟값 | 불가 | O(log n) |
| 코드량 | ~20줄 | ~60줄 |
| 상수(실제 속도) | 매우 작음 | 보통 |
| 메모리 | n+1 | 4n |

*차분 배열(difference array) 기법을 적용한 변형

**결론**: 구간 합만 필요하면 Fenwick, 최솟값/최댓값이나 복잡한 연산이 필요하면 Segment Tree.

## 변형: Range Update + Point Query

기본 Fenwick은 "점 갱신 + 구간 질의"인데, **차분 배열** 아이디어로 뒤집을 수 있다.

```
차분 배열 diff[i] = arr[i] - arr[i-1]
→ arr[i] = diff[0] + diff[1] + ... + diff[i] = prefixSum(i)

"arr[2..5] += 3"은:
  diff[2] += 3
  diff[6] -= 3
→ 점 갱신 2번으로 구간 갱신 완료!
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| Build (fromArray) | O(n) |
| 점 갱신 (update) | O(log n) |
| Prefix Sum | O(log n) |
| 구간 합 (rangeSum) | O(log n) |
| 공간 | O(n) |

## 실무에서의 활용

### 1. 실시간 순위/랭킹 시스템
게임 리더보드에서 "내 점수보다 높은 사람이 몇 명?"을 빠르게 계산.
점수를 인덱스로 사용해 Fenwick에 저장하면 prefix sum으로 순위를 O(log n)에 구한다.

### 2. 역전 쌍(Inversion Count) 계산
배열에서 i < j인데 arr[i] > arr[j]인 쌍의 수.
정렬 알고리즘의 성능 분석, 추천 시스템(사용자 선호도 유사도 측정)에 사용.
Fenwick으로 O(n log n)에 계산.

### 3. 누적 빈도 테이블
압축 알고리즘(Arithmetic Coding)에서 심볼의 누적 빈도를 유지.
심볼이 나올 때마다 빈도를 갱신하고 누적 분포를 빠르게 질의.

### 4. 2D Fenwick Tree
2차원 배열의 부분 합. 이미지 처리에서 "이 직사각형 영역의 픽셀 합"을 빠르게 계산.
Summed Area Table의 동적 버전.
