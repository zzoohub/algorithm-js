# Stack (스택)

## 개념

**LIFO (Last In, First Out)** — 마지막에 넣은 것이 먼저 나온다.

접시를 쌓는 것과 같다. 맨 위에 올리고(push), 맨 위에서 꺼낸다(pop).

```
push(3) → [1, 2, 3]
pop()   → [1, 2]  (3이 나옴)
peek()  → 2       (꺼내지 않고 확인만)
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| push | O(1) |
| pop | O(1) |
| peek | O(1) |

## 동작 원리

배열의 끝을 top으로 사용한다. `push`는 배열 끝에 추가, `pop`은 배열 끝에서 제거.
연결 리스트로도 구현할 수 있지만, JS에서는 배열이 더 간단하고 캐시 효율도 좋다.

## 실무에서의 활용

### 1. 브라우저 뒤로가기/앞으로가기
```
방문: A → B → C
뒤로가기 스택: [A, B, C]  ← pop하면 C에서 B로
앞으로가기 스택: [C]      ← 뒤로 간 페이지 push
```

### 2. 함수 호출 스택 (Call Stack)
프로그램이 함수를 호출하면 스택 프레임이 쌓인다. 재귀가 너무 깊으면 "Stack Overflow"가 발생하는 이유.
```
main() → foo() → bar()
콜스택: [main, foo, bar]  ← bar가 끝나면 pop → foo로 돌아감
```

### 3. Undo/Redo (실행 취소)
에디터에서 Ctrl+Z를 누르면 최근 작업부터 취소된다 — 스택.

### 4. 괄호 매칭
컴파일러가 `{[()]}` 같은 괄호 쌍을 검증할 때 스택을 사용한다.

### 5. DFS (깊이 우선 탐색)
재귀 대신 명시적 스택을 사용해서 DFS를 구현할 수 있다.
