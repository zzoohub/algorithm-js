# Graph (그래프)

## 개념

**정점(Vertex)**과 **간선(Edge)**으로 구성된 자료구조.
트리도 그래프의 일종이지만, 그래프는 사이클이 있을 수 있고 부모-자식 관계가 없다.

```
무방향 그래프:          방향 그래프(Directed):
  A --- B               A → B
  |   / |               ↑   ↓
  |  /  |               D ← C
  C --- D
```

## 표현 방식

### 인접 리스트 (이 구현)
```ts
Map {
  "A" => [{ node: "B", weight: 1 }, { node: "C", weight: 1 }],
  "B" => [{ node: "A", weight: 1 }, { node: "C", weight: 1 }],
  ...
}
```
- 메모리: O(V + E)
- 간선 존재 확인: O(degree)
- **희소 그래프에 유리** (대부분의 실제 그래프)

### 인접 행렬
```
    A  B  C  D
A [ 0, 1, 1, 0 ]
B [ 1, 0, 1, 1 ]
C [ 1, 1, 0, 1 ]
D [ 0, 1, 1, 0 ]
```
- 메모리: O(V²)
- 간선 존재 확인: O(1)
- **밀집 그래프에 유리**

## 그래프의 종류
- **무방향 그래프**: 간선에 방향이 없음 (친구 관계)
- **방향 그래프 (DAG 포함)**: 간선에 방향이 있음 (팔로우 관계)
- **가중치 그래프**: 간선에 비용이 있음 (지도에서 거리)

## 실무에서의 활용

### 1. 소셜 네트워크
사용자 = 정점, 친구/팔로우 = 간선.
"친구의 친구 추천" = BFS로 2-hop 이내 탐색.

### 2. 의존성 관리
npm 패키지 의존성, webpack 모듈 그래프, 빌드 시스템(Makefile)이 DAG.
순환 의존성 감지 = 사이클 탐지.

### 3. 네트워크 라우팅
라우터 = 정점, 회선 = 간선, 대역폭 = 가중치.
최단 경로 알고리즘으로 패킷 라우팅.

### 4. 추천 시스템
상품/사용자를 그래프로 연결해서 유사 사용자가 구매한 상품을 추천.

### 5. 지도/네비게이션
교차로 = 정점, 도로 = 간선, 거리/시간 = 가중치.
Dijkstra / A*로 최단 경로 계산.

### 6. Git
커밋 히스토리가 DAG이다. merge는 두 정점이 하나로 합쳐지는 것.
