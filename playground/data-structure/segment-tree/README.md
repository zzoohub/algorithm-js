# Segment Tree (세그먼트 트리)

## 개념

배열의 **구간 질의(range query)**와 **갱신(update)**을 모두 **O(log n)**에 처리하는 이진 트리.
"구간 합", "구간 최솟값", "구간 최댓값" 등 **결합 가능한(associative) 연산**이면 뭐든 된다.

```
원본 배열: [1, 3, 5, 7, 9, 11]

세그먼트 트리 (구간 합):

                [36]              ← 전체 합 (0~5)
             /        \
          [9]          [27]       ← (0~2), (3~5)
         /   \        /    \
       [4]   [5]   [16]   [11]   ← (0~1), (2), (3~4), (5)
      / \          / \
    [1] [3]     [7] [9]          ← 리프 = 원본 값
```

## 왜 필요한가?

| 연산 | 일반 배열 | Prefix Sum | Segment Tree |
|------|-----------|------------|--------------|
| 구간 합 질의 | O(n) | O(1) | O(log n) |
| 점 갱신 | O(1) | O(n) 재계산 | O(log n) |
| 구간 갱신 | O(n) | O(n) | O(log n) + Lazy |

Prefix Sum은 질의는 빠르지만 값이 바뀌면 전체를 다시 계산해야 한다.
Segment Tree는 **질의와 갱신 모두** 균형 있게 빠르다.

## 동작 원리

### Build — O(n)
배열을 리프에 넣고, 부모는 자식 두 개를 합쳐서 올라간다.

### Query(left, right) — O(log n)
루트에서 내려가며 세 가지 경우를 처리:
1. **현재 구간이 질의 범위 밖** → 0 반환 (기여 없음)
2. **현재 구간이 질의 범위에 완전히 포함** → 저장된 값 반환
3. **일부만 겹침** → 자식에게 위임, 결과 합침

```
query(1, 4): 인덱스 1~4의 합을 구하라

          [36]         → 일부 겹침, 자식에게
         /     \
      [9]      [27]    → [9]는 일부 겹침, [27]는 일부 겹침
     /   \    /    \
   [4]  [5] [16]  [11] → [4]는 일부, [5]는 포함, [16]는 포함, [11]은 밖
   / \       / \
 [1][3]   [7][9]       → [3]은 포함

결과: 3 + 5 + 16 = 24  (= 3 + 5 + 7 + 9)
```

### Update(index, value) — O(log n)
리프를 찾아가서 값을 바꾸고, 루트까지 올라오며 부모를 갱신.

## Lazy Propagation (지연 전파)

### 문제
"인덱스 0~1000에 모두 +5" 같은 **구간 갱신**을 점 갱신으로 하면 O(n log n).

### 아이디어
갱신을 **즉시 하지 않고 노드에 메모**해둔다.
나중에 그 구간을 읽을 때 비로소 자식에게 전파(propagate).

```
rangeUpdate(0, 5, +3):  전체에 3을 더하라

  [36] lazy=0           [36+18=54] lazy=0
     ↓ 구간 완전 포함이므로    → tree에 3×6=18 더하고
                              lazy=3 기록하고 멈춤
                              (자식은 아직 모름!)

나중에 query(0, 2)가 오면:
  [54] lazy=0 → 자식에 전파 필요
  왼쪽 자식: tree += 3×3=9, lazy=3
  오른쪽 자식: tree += 3×3=9, lazy=3
  현재 lazy = 0 (해소)
```

이 "필요할 때까지 미루기"가 핵심이고, OS의 copy-on-write, React의 batch update와 같은 사고 패턴이다.

## 시간 복잡도

| 연산 | 기본 | + Lazy Propagation |
|------|------|--------------------|
| Build | O(n) | O(n) |
| 점 갱신 | O(log n) | — |
| 구간 갱신 | O(n log n) | **O(log n)** |
| 구간 질의 | O(log n) | O(log n) |
| 공간 | O(n) | O(n) |

## 실무에서의 활용

### 1. 데이터베이스 Range Query
"가격이 100~500인 상품의 합계" 같은 범위 질의. B-Tree 인덱스의 내부 동작 원리가 세그먼트 트리와 유사하다.

### 2. 실시간 통계 대시보드
시계열 데이터에서 "최근 1시간의 합/최대/평균"을 실시간으로 계산.
데이터가 계속 추가/수정되는 환경에서 매번 O(n) 재계산하지 않아도 된다.

### 3. 게임 엔진의 충돌 감지
2D/3D 공간을 구간으로 나눠서 "이 범위에 객체가 몇 개?"를 빠르게 질의.

### 4. 네트워크 트래픽 모니터링
시간대별 패킷 수를 세그먼트 트리에 저장, 특정 시간 범위의 합계를 빠르게 질의.

### 5. 경쟁 프로그래밍
구간 합, 구간 최솟값(RMQ), 구간 GCD 등 대부분의 "구간 + 갱신" 문제에 적용.
