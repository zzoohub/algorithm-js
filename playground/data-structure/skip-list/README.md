# Skip List (스킵 리스트)

## 개념

정렬된 연결 리스트에 **여러 레벨의 "고속도로"**를 추가해서
O(log n) 검색을 가능하게 한 확률적 자료구조.

균형 BST(AVL, Red-Black)의 대안으로, 구현이 훨씬 간단하다.

```
Level 3: header ────────────────────→ 25 ─────→ null
Level 2: header ──→ 6 ──────────────→ 25 ─────→ null
Level 1: header ──→ 6 ──→ 9 ────────→ 25 → 30 → null
Level 0: header → 3 → 6 → 7 → 9 → 12 → 25 → 30 → null
```

## 시간 복잡도

| 연산 | 평균 | 최악 |
|------|------|------|
| 검색 | O(log n) | O(n) |
| 삽입 | O(log n) | O(n) |
| 삭제 | O(log n) | O(n) |

확률적으로 균형을 유지하므로, 최악의 경우는 극히 드물다.

## 동작 원리

### 레벨 결정 (동전 던지기)
새 노드를 삽입할 때, "동전"을 던져서 레벨을 결정한다.
- 앞면 (50%) → 한 레벨 올림
- 뒷면 → 멈춤
- 평균적으로 레벨 1, 50%가 레벨 0, 25%가 레벨 1, 12.5%가 레벨 2...

### 검색
가장 높은 레벨에서 시작해서 오른쪽으로 이동하다가
찾는 값보다 커지면 한 레벨 아래로 내려간다.
```
찾기: 9
Level 2: header → 6 → (25는 9보다 큼, 아래로)
Level 1: header → 6 → 9 ✓ 찾음!
```

### 삽입
1. 각 레벨에서 삽입 위치의 이전 노드를 기록 (update 배열)
2. 랜덤 레벨 결정
3. 각 레벨에서 새 노드를 삽입 (연결 리스트 삽입)

### 왜 균형 트리 대신 사용하는가?
- AVL/Red-Black 트리: 회전 연산이 복잡, 동시성 처리가 어려움
- 스킵 리스트: 구현이 간단, 락 없는(lock-free) 버전이 쉬움

## 실무에서의 활용

### 1. Redis의 Sorted Set (ZSET)
Redis의 가장 중요한 자료구조 중 하나인 Sorted Set이 스킵 리스트로 구현됨.
`ZADD`, `ZRANGE`, `ZRANK` 등의 명령어가 스킵 리스트에서 동작.

왜 Red-Black 트리가 아닌가?
- 범위 쿼리가 더 효율적 (ZRANGEBYSCORE)
- 구현이 단순하고 디버깅이 쉬움
- 동시성 제어가 용이

### 2. LevelDB / RocksDB의 MemTable
Google의 LevelDB가 메모리 내 정렬 테이블(MemTable)로 스킵 리스트를 사용.

### 3. Java의 ConcurrentSkipListMap
멀티스레드 환경에서 정렬된 맵이 필요할 때 사용.
Red-Black 트리보다 동시성 성능이 좋다.

### 4. Lucene (검색 엔진)
Apache Lucene의 인메모리 인덱스에서 스킵 리스트를 활용.
