# Heap (힙)

## 개념

**완전 이진 트리** 기반으로, 부모가 항상 자식보다 우선순위가 높은 자료구조.
- **Min Heap**: 부모 <= 자식 (루트가 최솟값)
- **Max Heap**: 부모 >= 자식 (루트가 최댓값)

```
Min Heap:
        1
       / \
      3    2
     / \
    7    4
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| insert | O(log n) |
| extract (최솟/최댓값 꺼내기) | O(log n) |
| peek (최솟/최댓값 확인) | O(1) |

## 동작 원리

### 배열로 트리를 표현
트리를 배열에 저장한다. 인덱스 i의 노드에 대해:
- 부모: `Math.floor((i - 1) / 2)`
- 왼쪽 자식: `2 * i + 1`
- 오른쪽 자식: `2 * i + 2`

```
배열: [1, 3, 2, 7, 4]
       0  1  2  3  4   ← 인덱스
```

### insert (Bubble Up)
1. 배열 맨 끝에 삽입
2. 부모와 비교해서 우선순위가 높으면 swap
3. 루트에 도달하거나 부모보다 우선순위가 낮을 때까지 반복

### extract (Bubble Down)
1. 루트(배열[0])를 꺼냄
2. 맨 마지막 요소를 루트로 이동
3. 자식과 비교해서 우선순위가 낮으면 더 높은 자식과 swap
4. 리프에 도달하거나 자식보다 우선순위가 높을 때까지 반복

## 실무에서의 활용

### 1. 우선순위 큐 (Priority Queue)
작업 스케줄러가 우선순위가 높은 작업을 먼저 처리한다.
OS의 프로세스 스케줄링, 네트워크 패킷 QoS 등.

### 2. Dijkstra 최단 경로 알고리즘
다음으로 방문할 최소 비용 노드를 O(log n)으로 꺼낸다.

### 3. 이벤트 드리븐 시뮬레이션
가장 빠른 시간의 이벤트부터 처리한다 (게임 엔진, 네트워크 시뮬레이터).

### 4. Top-K 문제
"가장 많이 검색된 단어 10개" — Min Heap에 10개만 유지하면 O(n log k).

### 5. 메디안 유지
Max Heap + Min Heap 두 개로 실시간 스트림에서 중앙값을 O(log n)으로 유지한다.
