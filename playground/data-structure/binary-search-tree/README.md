# Binary Search Tree (이진 탐색 트리)

## 개념

모든 노드가 **왼쪽 < 부모 < 오른쪽** 규칙을 만족하는 이진 트리.
정렬된 데이터에서 빠르게 검색/삽입/삭제할 수 있다.

```
        8
       / \
      3    10
     / \     \
    1    6    14
        / \   /
       4   7 13
```

## 시간 복잡도

| 연산 | 평균 | 최악 (편향 트리) |
|------|------|------|
| 검색 | O(log n) | O(n) |
| 삽입 | O(log n) | O(n) |
| 삭제 | O(log n) | O(n) |

최악의 경우: 1, 2, 3, 4, 5 순서로 삽입하면 한쪽으로 치우친 연결 리스트가 됨.
→ 이를 해결한 것이 AVL 트리, Red-Black 트리.

## 동작 원리

### 검색
루트부터 시작해서 찾는 값이 현재 노드보다 작으면 왼쪽, 크면 오른쪽으로 이동.
매 단계마다 탐색 범위가 절반으로 줄어든다.

### 삽입
검색과 같은 경로를 따라 내려가다가 빈 자리에 삽입.

### 삭제 (가장 복잡)
1. **리프 노드**: 그냥 제거
2. **자식 하나**: 자식을 삭제할 노드 자리에 올림
3. **자식 둘**: 오른쪽 서브트리의 최솟값(successor)으로 대체 후 successor 삭제

### 순회
- **중위 (Inorder)**: 왼→루트→오른 = 정렬된 순서
- **전위 (Preorder)**: 루트→왼→오른 = 트리 구조 저장/복원
- **후위 (Postorder)**: 왼→오른→루트 = 자식 먼저 처리 (삭제 등)

## 실무에서의 활용

### 1. 데이터베이스 인덱스 (B-Tree)
DB의 인덱스는 BST를 일반화한 B-Tree/B+Tree이다.
`WHERE id > 100 AND id < 200` 같은 범위 쿼리가 빠른 이유.

### 2. 파일 시스템
ext4, NTFS 등이 디렉토리 엔트리를 B-Tree로 관리한다.

### 3. Map/Set의 내부 구현
C++의 `std::map`, Java의 `TreeMap`이 Red-Black Tree (자가 균형 BST).

### 4. 자동완성/사전
정렬된 단어 목록에서 범위 검색을 빠르게 수행.
