# Tree (트리)

## 개념

**계층적 구조**를 표현하는 비선형 자료구조.
하나의 루트 노드에서 시작해 자식 노드들이 가지처럼 뻗어나간다.

```
       A          ← 루트
      /|\
     B  C  D      ← A의 자식
    / \    |
   E   F   G     ← 리프 (자식이 없는 노드)
```

- **루트 (Root)**: 최상위 노드
- **리프 (Leaf)**: 자식이 없는 노드
- **깊이 (Depth)**: 루트에서 해당 노드까지의 거리
- **높이 (Height)**: 해당 노드에서 가장 깊은 리프까지의 거리

## 동작 원리

### DFS (깊이 우선 탐색)
한 가지를 끝까지 탐색한 후 다음 가지로 이동한다.
재귀 또는 스택으로 구현.
```
탐색 순서: A → B → E → F → C → D → G
```

### BFS (너비 우선 탐색)
같은 레벨을 먼저 탐색한 후 다음 레벨로 내려간다.
큐로 구현.
```
탐색 순서: A → B → C → D → E → F → G
```

## 실무에서의 활용

### 1. DOM (Document Object Model)
HTML 문서가 트리 구조이다.
```
<html>
  <body>
    <div>
      <p>Hello</p>
    </div>
  </body>
</html>
```

### 2. 파일 시스템
디렉토리와 파일의 계층 구조가 트리이다.

### 3. React 컴포넌트 트리
컴포넌트가 부모-자식 관계로 중첩된다. 리렌더링이 서브트리 단위로 일어난다.

### 4. AST (Abstract Syntax Tree)
컴파일러/트랜스파일러(Babel, TypeScript)가 코드를 파싱하면 트리가 된다.
ESLint의 규칙들도 AST를 탐색하면서 동작한다.

### 5. 조직도, 댓글 답글, 카테고리 분류
계층적 데이터는 전부 트리로 모델링된다.
