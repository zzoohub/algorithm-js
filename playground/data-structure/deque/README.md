# Deque (덱, Double-Ended Queue)

## 개념

**양쪽 끝에서 삽입과 삭제가 모두 가능**한 자료구조.
스택과 큐를 합친 것이라고 볼 수 있다.

```
pushFront ← [front ... back] → pushBack
popFront  ← [front ... back] → popBack
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| pushFront | O(1) |
| pushBack | O(1) |
| popFront | O(1) |
| popBack | O(1) |

## 동작 원리

이중 연결 리스트(Doubly Linked List)로 구현한다.
각 노드가 prev와 next를 가지므로 양쪽 끝에서 O(1) 연산이 가능하다.

배열로 구현하면 앞쪽 삽입/삭제가 O(n)이 되므로 연결 리스트가 적합하다.
(C++의 std::deque는 청크 배열로 구현해서 배열의 캐시 이점도 살린다)

## 실무에서의 활용

### 1. 슬라이딩 윈도우 최대/최소값
크기 k인 윈도우가 배열을 지나가면서 각 위치의 최대값을 O(n)에 구한다.
덱의 앞쪽에는 항상 현재 윈도우의 최대값 인덱스가 위치하도록 유지.
```
배열: [1, 3, -1, -3, 5, 3, 6, 7], k=3
윈도우 최대값: [3, 3, 5, 5, 6, 7]
```

### 2. 0-1 BFS
간선 가중치가 0 또는 1인 그래프에서 최단 경로를 구할 때,
가중치 0이면 앞에 넣고(pushFront), 1이면 뒤에 넣어서(pushBack)
Dijkstra 없이 O(V+E)에 해결한다.

### 3. 작업 스틸링 (Work Stealing)
멀티스레드 스케줄러에서 각 스레드가 자기 덱의 한쪽에서 작업을 처리하고,
놀고 있는 스레드가 다른 스레드의 반대쪽에서 작업을 가져간다.
Go 런타임, Java ForkJoinPool이 이 방식을 사용한다.

### 4. 브라우저 히스토리
뒤로가기/앞으로가기가 덱의 양쪽 끝 접근과 유사하다.
