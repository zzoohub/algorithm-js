# Interval Tree (구간 트리)

## 개념

**"이 시간에 겹치는 회의가 뭐가 있지?"** 같은 질문에 빠르게 답하는 자료구조.
구간(interval)들의 집합에서 특정 점이나 구간과 겹치는 모든 구간을 **O(log n + k)**에 찾는다.

```
구간들: [9,12] [15,20] [10,14] [13,17] [5,8]

"시간 11에 진행 중인 것은?"
→ [9,12], [10,14]  ← O(log n + k)로 찾음

"[11,16]과 겹치는 것은?"
→ [9,12], [15,20], [10,14], [13,17]
```

## 왜 필요한가?

| 방법 | 겹침 질의 | 삽입 |
|------|-----------|------|
| 배열 전체 순회 | O(n) | O(1) |
| 정렬 + 이진 탐색 | O(n) 최악 | O(n) |
| **Interval Tree** | **O(log n + k)** | **O(log n)** |

구간이 수만 개일 때 매번 전체를 순회하면 느리다. Interval Tree는 BST 구조에 **max 값**이라는 한 줄의 추가 정보만으로 대부분의 서브트리를 가지치기한다.

## 동작 원리

### 구조: BST + max 보강(augmentation)

```
BST 기준: 구간의 시작점(low)으로 정렬
max: 해당 서브트리 내 모든 구간의 끝점(high) 중 최댓값

        [15,20] max=40
       /              \
  [10,30] max=30    [17,19] max=40
  /      \                 \
[5,20]  [12,15]          [30,40]
max=20  max=15           max=40
```

### 겹침 검사
두 구간 [a, b]와 [c, d]가 겹치는 조건:
```
a ≤ d  AND  c ≤ b

[---a---b---]
      [---c---d---]   ← 겹침: a ≤ d ✓, c ≤ b ✓

[---a---b---]
              [---c---d---]  ← 안 겹침: a ≤ d ✓, c ≤ b ✗
```

### 검색 알고리즘

1. 현재 노드의 구간이 질의와 겹치면 → 결과에 추가
2. 왼쪽 서브트리의 **max ≥ query.low** → 왼쪽에 겹치는 게 있을 수 있으니 탐색
3. 오른쪽 서브트리에도 겹칠 가능성이 있으면 탐색

```
searchAll([11, 16]):

[15,20] max=40  → 겹침 ✓ (11≤20, 15≤16)
├─ 왼쪽 max=30 ≥ 11 → 탐색
│  [10,30] → 겹침 ✓
│  ├─ [5,20] → 겹침 ✓
│  └─ [12,15] → 겹침 ✓
└─ 오른쪽
   [17,19] → 겹침 ✗ (17 > 16)
   └─ [30,40] → 겹침 ✗
```

**max로 가지치기**: 왼쪽 서브트리의 max가 query.low보다 작으면, 왼쪽에는 겹치는 구간이 절대 없다. 서브트리 전체를 건너뛴다.

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| 삽입 | O(log n) |
| 겹치는 구간 1개 찾기 | O(log n) |
| 겹치는 모든 구간 찾기 | O(log n + k) |
| 특정 시점 질의 | O(log n + k) |
| 공간 | O(n) |

k = 결과 개수. 겹치는 게 많으면 당연히 오래 걸리지만, 그건 출력 자체의 크기이므로 최적이다.

## 실무에서의 활용

### 1. 캘린더 / 회의실 예약
"이 시간대에 비어있는 회의실은?" → 구간이 겹치지 않는 것을 찾기.
Google Calendar, Outlook이 내부적으로 유사한 구조를 사용.

### 2. 데이터베이스 Range Lock
트랜잭션이 "행 100~200에 락을 건다" → 새 트랜잭션이 "행 150~250에 접근"하면 충돌.
InnoDB의 gap lock 관리에 구간 검색이 필요.

### 3. 게놈 분석 (Bioinformatics)
유전자 위치가 구간으로 표현됨. "염색체 3번의 15000~20000 위치에 있는 유전자는?"
BEDTools 같은 도구가 interval tree 기반.

### 4. 네트워크 라우팅
IP 주소 범위(CIDR 블록)를 구간으로 관리. 특정 IP가 어떤 서브넷에 속하는지 빠르게 판별.

### 5. 게임 엔진 충돌 감지
객체의 바운딩 박스를 구간으로 표현. 각 축(x, y)에 대해 interval tree를 유지하면 O(n)보다 훨씬 빠르게 충돌 후보를 좁힐 수 있다.
