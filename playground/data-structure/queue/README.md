# Queue (큐)

## 개념

**FIFO (First In, First Out)** — 먼저 넣은 것이 먼저 나온다.

줄 서기와 같다. 뒤에서 들어가고(enqueue), 앞에서 나간다(dequeue).

```
enqueue(3) → [1, 2, 3]
dequeue()  → [2, 3]  (1이 나옴)
```

## 시간 복잡도

| 연산 | 복잡도 |
|------|--------|
| enqueue | O(1) |
| dequeue | O(1) |
| peek | O(1) |

## 동작 원리

연결 리스트로 구현한다. head에서 dequeue, tail에서 enqueue.
배열의 `shift()`는 O(n)이라 큐에는 부적합하다 (모든 요소를 한 칸씩 당겨야 함).

## 실무에서의 활용

### 1. 작업 큐 / 메시지 큐
서버에서 요청이 들어오면 큐에 넣고 순서대로 처리한다.
- RabbitMQ, AWS SQS, Bull(Redis 기반) 등이 전부 큐

### 2. BFS (너비 우선 탐색)
트리/그래프를 레벨 단위로 탐색할 때 큐를 사용한다.

### 3. 프린터 대기열
문서를 보낸 순서대로 출력한다.

### 4. 이벤트 루프
Node.js의 이벤트 루프에서 콜백 함수들이 큐(태스크 큐, 마이크로태스크 큐)에 쌓여서 순서대로 실행된다.

### 5. Rate Limiting
API 요청을 큐에 넣고 일정 속도로만 처리해서 트래픽을 제어한다.
