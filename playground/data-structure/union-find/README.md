# Union-Find (Disjoint Set, 서로소 집합)

## 개념

**"이 두 요소가 같은 그룹인가?"**를 매우 빠르게 판단하는 자료구조.
두 가지 핵심 연산:
- **Find**: 어떤 요소가 속한 그룹의 대표(루트)를 찾는다
- **Union**: 두 그룹을 합친다

```
초기: {0} {1} {2} {3} {4}  (각자 독립)
union(0, 1) → {0,1} {2} {3} {4}
union(2, 3) → {0,1} {2,3} {4}
union(0, 3) → {0,1,2,3} {4}
connected(1, 2) → true  (같은 그룹)
connected(1, 4) → false (다른 그룹)
```

## 시간 복잡도

두 가지 최적화를 적용하면 사실상 O(1):

| 연산 | 복잡도 |
|------|--------|
| find | O(α(n)) ≈ O(1) |
| union | O(α(n)) ≈ O(1) |
| connected | O(α(n)) ≈ O(1) |

α(n)은 역 아커만 함수로, 실질적으로 4를 넘지 않는다.

## 동작 원리

### 기본 구조
각 요소가 부모를 가리키는 배열. 루트는 자기 자신을 가리킨다.
```
parent: [0, 0, 2, 2, 4]
         ↑ 0의 부모=0 (루트)
            ↑ 1의 부모=0
```

### 최적화 1: Path Compression (경로 압축)
find할 때 거치는 모든 노드를 루트에 직접 연결한다.
```
Before: 0 ← 1 ← 2 ← 3
After:  0 ← 1, 0 ← 2, 0 ← 3   (모두 루트에 직접 연결)
```

### 최적화 2: Union by Rank
높이가 낮은 트리를 높은 트리 아래에 붙여서 트리가 한쪽으로 치우치지 않게 한다.

## 실무에서의 활용

### 1. 네트워크 연결 확인
"서버 A에서 서버 B로 연결이 가능한가?" — 네트워크 토폴로지를 Union-Find로 관리.

### 2. 크루스칼 최소 신장 트리 (MST)
간선을 비용 순으로 정렬한 후, 사이클이 생기지 않는 간선만 추가.
"사이클이 생기는가?" = "두 정점이 이미 connected인가?"

### 3. 이미지 처리 — Connected Components
인접한 같은 색상의 픽셀을 그룹으로 묶을 때 (포토샵의 마술봉 도구).

### 4. 소셜 네트워크 친구 그룹
"A와 B가 (직/간접적으로) 연결되어 있는가?"

### 5. 코딩 테스트
섬의 개수, 네트워크 연결, 집합 합치기 등 매우 자주 출제되는 패턴.
