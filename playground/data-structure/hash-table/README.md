# Hash Table (해시 테이블)

## 개념

**키(key)를 해시 함수로 변환**해서 배열의 인덱스로 사용하는 자료구조.
JS의 `Map`, `Object`, Python의 `dict`가 모두 해시 테이블 기반이다.

```
"name" → hash("name") → 17 → buckets[17] = "Alice"
"age"  → hash("age")  → 3  → buckets[3]  = 25
```

## 시간 복잡도

| 연산 | 평균 | 최악 (모든 키가 충돌) |
|------|------|------|
| get | O(1) | O(n) |
| set | O(1) | O(n) |
| delete | O(1) | O(n) |

## 동작 원리

### 해시 함수
키 문자열을 숫자로 변환한다. 좋은 해시 함수는 **균일 분포**를 만든다.
```
hash("abc") = (97*31^2 + 98*31 + 99) % 53 = 결과 인덱스
```

### 충돌 해결
두 키가 같은 인덱스에 매핑되면 "충돌"이 발생한다.
- **체이닝 (Chaining)**: 같은 버킷에 연결 리스트/배열로 저장 ← 이 구현이 사용하는 방식
- **개방 주소법 (Open Addressing)**: 다음 빈 슬롯을 찾아서 저장

### 소수를 사용하는 이유
버킷 크기를 소수(53)로 하면 해시값이 더 균일하게 분포된다.
합성수면 특정 패턴의 키가 같은 버킷에 몰리기 쉽다.

## 실무에서의 활용

### 1. JavaScript의 Object와 Map
`{}`, `new Map()` 모두 내부적으로 해시 테이블이다.

### 2. 데이터베이스 인덱스
Hash Index — 정확한 키 매칭이 필요할 때 B-Tree 대신 사용한다.

### 3. 캐싱 시스템
Redis, Memcached가 키-값 기반으로 동작한다.

### 4. 중복 검사
이미 처리한 요청인지, 이미 방문한 URL인지 O(1)으로 확인한다.

### 5. 라우팅 테이블
웹 서버가 URL 패턴을 핸들러 함수에 매핑할 때 해시 테이블을 사용한다.
