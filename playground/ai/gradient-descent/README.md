# 경사 하강법 (Gradient Descent)

## 개념

**"산에서 눈을 감고 가장 낮은 곳을 찾는 방법: 발밑의 경사를 느끼고, 내리막 방향으로 한 걸음씩 이동한다."**

이것이 신경망 학습의 전부다. 손실 함수라는 "산"에서 최저점을 찾는 것.

```
w_new = w_old - lr × ∂L/∂w
                ↑        ↑
           학습률    기울기(gradient)
```

## Learning Rate (학습률)

```
lr이 적절할 때:        lr이 너무 클 때:       lr이 너무 작을 때:
  ╲                     ╱╲  ╱╲               ╲
    ╲                  ╱    ╲   ╲ ← 발산!       ╲
      ╲              ╱           ╲                 ╲
        ╲╱ ← 수렴!                                   ╲ ← 너무 느림
```

실무에서 학습률은 가장 중요한 하이퍼파라미터. 보통 0.001 ~ 0.01로 시작.

## 변형들

### 1. Vanilla SGD (기본)

```
w = w - lr × gradient

문제: 기울기가 진동하면 수렴이 느림

    ↗ ↙ ↗ ↙ ↗ ↙  → 지그재그로 느리게 수렴
```

### 2. SGD + Momentum (모멘텀)

```
v = 0.9 × v_prev + 0.1 × gradient    ← "관성" 축적
w = w - lr × v

공이 굴러가는 것처럼:
- 같은 방향으로 계속 가면 → 가속
- 방향이 바뀌면 → 감속 (진동 억제)

    → → → → → →  ← 모멘텀이 진동을 억제하고 빠르게 수렴
```

### 3. Adam — 현대 딥러닝의 기본

```
m = β₁ × m + (1-β₁) × g        ← 평균 기울기 (어디로?)
v = β₂ × v + (1-β₂) × g²       ← 기울기 분산 (얼마나 변동?)

w = w - lr × m / (√v + ε)       ← 변동이 큰 차원은 작게,
                                    변동이 작은 차원은 크게
```

**왜 Adam이 좋은가:**
- 파라미터마다 학습률을 자동 조정
- Momentum의 가속 효과 포함
- 하이퍼파라미터 튜닝이 덜 까다로움
- "일단 Adam 쓰면 대부분 잘 됨"

## 배치 크기와 SGD의 관계

```
Batch Gradient Descent:    전체 데이터로 기울기 계산 → 1번 업데이트
  - 정확하지만 느림, 메모리 많이 필요

Stochastic GD (SGD):       1개 데이터로 기울기 계산 → 1번 업데이트
  - 빠르지만 불안정 (노이즈가 많음)

Mini-batch SGD:             32~256개 데이터로 기울기 계산 → 1번 업데이트
  - 실무의 표준. 속도와 안정성의 균형
  - GPU의 병렬 연산을 활용할 수 있는 크기
```

## 수렴 시각화

```
2D 손실 함수 위에서의 이동 경로:

       Vanilla SGD          Momentum             Adam
       ╭─╮╭─╮               ╭───╮
      │   ╲╱ │             │      ╲              ╭──╮
      │   ╱╲ │             │        ╲           │    ╲
       ╰─╯╰─⊛              ╰────────⊛           ╰────⊛
       (진동)            (직선 가속)          (적응적 수렴)
```

## Epoch, Batch, Iteration

```
데이터 1000개, batch_size = 100:

1 epoch = 전체 데이터를 1번 봄
        = 1000 / 100 = 10 iterations (10번의 가중치 업데이트)

보통 수십~수백 epoch을 학습한다.
```

## 실무에서

```python
# PyTorch에서의 실제 코드
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(100):
    for batch in dataloader:
        loss = model(batch)         # 순전파
        loss.backward()             # 역전파 (기울기 계산)
        optimizer.step()            # 가중치 업데이트 (경사 하강)
        optimizer.zero_grad()       # 기울기 초기화
```

이 4줄이 경사 하강법의 전부. 하지만 내부에서 일어나는 일을 이해해야 디버깅과 튜닝이 가능하다.
