# KNN (K-Nearest Neighbors)

## 개념

**"가장 가까운 K명의 이웃이 뭐라고 하는지 보고, 다수결로 결정한다."**
ML에서 가장 직관적인 알고리즘. 학습이란 게 없고, 데이터를 기억하는 것이 전부.

```
K=3일 때 ★의 분류:

  ● ●
    ● ★ ▲        ← ★ 주변 3개: ● 2개, ▲ 1개
      ▲ ▲           → 다수결: ● (원)으로 분류!
        ▲
```

## 동작

```
1. 학습: 데이터를 그냥 저장 (Lazy Learning)

2. 예측:
   a. 새 점과 모든 기존 데이터의 거리 계산
   b. 가장 가까운 K개 선택
   c. 다수결 (분류) 또는 평균 (회귀)
```

## K의 선택

```
K=1: 가장 가까운 1개만 봄           K=7: 7개를 봄
  → 노이즈에 민감 (과적합)          → 경계가 부드러움 (과소적합 위험)

  ●●●│▲▲▲                         ●●●    ▲▲▲
  ●●│▲│●▲  ← 울퉁불퉁              ●●●───▲▲▲  ← 매끄러움
  ●●│▲▲▲▲
```

**규칙**: K는 보통 홀수로 (동점 방지), √n 근처에서 시작.

## 거리 함수

```
유클리드 거리: √(Σ(aᵢ - bᵢ)²)  — 가장 일반적
맨해튼 거리:   Σ|aᵢ - bᵢ|      — 고차원에서 유리할 수 있음
코사인 유사도:  a·b / (|a||b|)   — 텍스트, 임베딩에서 주로 사용
```

## 차원의 저주 (Curse of Dimensionality)

KNN의 가장 큰 약점. 차원이 높아지면 "가까움"이 의미를 잃는다.

```
1차원: 100개의 점 → 이웃이 가까움
10차원: 100개의 점 → 데이터가 공간에 흩어져서 모든 점이 비슷하게 "멀다"
1000차원: 가장 가까운 점과 가장 먼 점의 거리 차이가 거의 없음!
```

해결: 차원 축소(PCA), 특성 선택, 또는 코사인 유사도 사용.

## KNN vs 신경망

| | KNN | 신경망 |
|---|---|---|
| 학습 | 없음 (저장만) | 있음 (가중치 최적화) |
| 예측 속도 | **느림** O(n×d) | **빠름** O(d) |
| 메모리 | 전체 데이터 저장 | 가중치만 저장 |
| 비선형 경계 | 가능 | 가능 |
| 해석 가능성 | **높음** ("이 3개가 비슷해서") | 낮음 |

KNN은 "왜 이렇게 예측했는가?"를 설명하기 쉽다 → **해석 가능한 AI**에서 가치.

## 실무에서의 활용

### 1. 추천 시스템 (Collaborative Filtering)
"비슷한 취향의 사용자 K명이 좋아한 영화" → 추천.
Netflix, Spotify의 초기 추천 알고리즘.

### 2. 이미지 분류 (베이스라인)
새로운 이미지와 가장 비슷한 학습 이미지 K개를 찾아 분류.
복잡한 모델 전에 **베이스라인(기준선)**으로 먼저 시도.

### 3. 이상치 탐지
K개 이웃까지의 평균 거리가 비정상적으로 크면 이상치.
금융 사기, 네트워크 침입 탐지.

### 4. 임베딩 검색 (Vector Search)
LLM의 임베딩 벡터에서 "가장 유사한 문서 K개" 찾기 = KNN.
Pinecone, Milvus 같은 벡터 DB가 근본적으로 KNN의 최적화된 버전.

### 5. 왜 알아야 하는가
KNN은 ML의 "Hello World"이면서 동시에 실무의 핵심 패턴:
- 벡터 검색 (RAG, 추천) = 대규모 KNN
- 임베딩 + 유사도 = 현대 AI의 근간
- "데이터가 곧 모델이다"라는 사고방식
