# 행렬 연산 (Matrix Operations)

## 왜 행렬인가?

신경망의 모든 계산은 결국 **행렬 곱셈 + 활성화 함수**의 반복이다.

```
뉴런 하나의 연산:
  output = f(w₁x₁ + w₂x₂ + w₃x₃ + b)
         = f(w · x + b)         ← 내적(dot product)

뉴런 여러 개 (레이어):
  outputs = f(W × inputs + b)   ← 행렬 곱셈
```

GPU가 AI에서 핵심인 이유: 행렬 곱셈은 **대량 병렬 연산**이고, GPU는 수천 개의 코어로 이걸 동시에 처리한다.

## 핵심 연산들

### 1. 행렬 곱셈 (MatMul) — 가장 중요

```
W × x = y

가중치 W:     입력 x:    출력 y:
[0.2  0.8]   [3]       [0.2×3 + 0.8×5]   [4.6]
[0.5  0.1] × [5]     = [0.5×3 + 0.1×5] = [2.0]
[0.9  0.4]             [0.9×3 + 0.4×5]   [4.7]

2개 입력 → 3개 뉴런 출력
(3×2) × (2×1) = (3×1)
```

신경망의 "학습"이란 이 W의 값을 조정하는 것이다.

### 2. 내적 (Dot Product) — 유사도 측정

```
a · b = Σ aᵢbᵢ

[1, 0, 1] · [1, 0, 1] = 2   ← 같은 방향 (유사)
[1, 0, 1] · [0, 1, 0] = 0   ← 직교 (무관)
[1, 0, 1] · [-1, 0, -1] = -2 ← 반대 방향 (비유사)
```

Transformer의 어텐션 스코어가 바로 내적: "이 단어와 저 단어가 얼마나 관련 있는가?"

### 3. 전치 (Transpose) — 역전파의 핵심

```
A = [1 2 3]     A^T = [1 4]
    [4 5 6]           [2 5]
                      [3 6]
```

순전파: `y = Wx`
역전파: `∂L/∂x = W^T × ∂L/∂y`  ← 기울기를 뒤로 전달할 때 전치 사용

### 4. 브로드캐스팅 — 편향(bias) 더하기

```
Wx + b:

[4.6]   [0.1]   [4.7]
[2.0] + [0.2] = [2.2]    ← 벡터 b를 각 행에 더함
[4.7]   [0.3]   [5.0]
```

배치(batch) 처리 시: 여러 입력에 같은 bias를 더하는 것이 브로드캐스팅.

### 5. 원소별 연산 (Element-wise)

```
활성화 함수 적용:
[4.7]        [4.7]
[2.2]  →ReLU→ [2.2]     ← 각 원소에 독립적으로 적용
[-0.3]       [0.0]
```

## 신경망에서의 전체 흐름

```
입력 x → [MatMul: W₁×x] → [Add: +b₁] → [ReLU] → [MatMul: W₂×h] → [Add: +b₂] → [Softmax] → 출력

        행렬 곱셈        브로드캐스팅   원소별      행렬 곱셈      브로드캐스팅   원소별
```

## 초기화가 왜 중요한가 (Xavier/He)

가중치를 너무 크게 초기화하면 값이 폭발하고, 너무 작으면 소실된다.
Xavier 초기화: `W ~ N(0, 1/√n)` — 입력 수에 따라 스케일 조정.

```
나쁜 초기화:  W = randn(3, 2, scale=10)  → 값 폭발 → NaN
좋은 초기화:  W = randn(3, 2, scale=1/√2) → 안정적 학습
```

## 실무에서

실제로는 이걸 직접 구현하지 않고 **NumPy, PyTorch, TensorFlow**가 해준다.
하지만 원리를 모르면:
- 차원 불일치 에러를 디버깅할 수 없다
- 모델 구조를 설계할 수 없다
- GPU 메모리 문제를 예측할 수 없다
