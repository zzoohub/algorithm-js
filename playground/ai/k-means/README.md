# K-Means 클러스터링

## 개념

**라벨(정답) 없이** 데이터를 K개의 그룹으로 자동 분류하는 **비지도 학습** 알고리즘.
"비슷한 것끼리 묶어라"를 수학적으로 수행한다.

```
입력: 라벨 없는 점들          출력: K=3 클러스터
  ·  ·                        ●  ●
    · ·                         ● ●
        ·  ·                       ▲  ▲
         ·                          ▲
  ·                           ■
    ·  ·                        ■  ■
```

## 알고리즘 (3단계 반복)

### 1단계: 초기화 — K개 중심점 배치

```
K=2, 랜덤 중심: ★₁(2,3), ★₂(8,7)

  · · ·           ★₁
    · · ·
                  ★₂
      · · ·
```

### 2단계: 할당 — 각 점을 가장 가까운 중심에 배정

```
  ● ● ●           ★₁         ← ★₁에 가까운 점들 → 클러스터 1
    ● ● ●
                  ★₂
      ▲ ▲ ▲                  ← ★₂에 가까운 점들 → 클러스터 2
```

### 3단계: 갱신 — 각 클러스터의 평균으로 중심 이동

```
  ● ● ●
    ●★₁● ●                   ← 중심이 클러스터 평균으로 이동
              ★₂
      ▲ ▲ ▲
```

### 반복 → 수렴

```
반복 1: 중심 이동 큼
반복 2: 중심 이동 작아짐
반복 3: 중심 거의 안 움직임 → 수렴! 종료
```

## K-Means++ 초기화

랜덤 초기화의 문제: 중심이 가까이 모이면 나쁜 결과.

```
나쁜 초기화:            K-Means++:
  ★₁★₂  ← 둘 다 왼쪽!     ★₁        ★₂  ← 멀리 분산!
```

**K-Means++**: 기존 중심과 **멀리 떨어진 점**을 높은 확률로 다음 중심으로 선택.
거리²에 비례하는 확률 → 자연스럽게 분산된 초기 중심.

## 지도 학습 vs 비지도 학습

| | 지도 학습 (Supervised) | 비지도 학습 (Unsupervised) |
|---|---|---|
| 데이터 | (입력, **정답**) 쌍 | 입력만 |
| 예시 | "이 이미지는 고양이" | "비슷한 이미지끼리 묶어라" |
| 알고리즘 | 신경망, KNN, SVM | **K-Means**, PCA, DBSCAN |
| 평가 | 정확도 (정답과 비교) | 실루엣 점수, 시각적 확인 |

## K를 어떻게 정하는가? — Elbow Method

```
K를 1부터 늘려가며 "클러스터 내 거리 합"을 측정:

거리
│╲
│  ╲
│    ╲___
│        ╲___________
│
└────────────────── K
  1  2  3  4  5  6

꺾이는 지점(elbow) = 적절한 K
위 예에서 K=3이 적절
```

## 시간 복잡도

| | 복잡도 |
|---|---|
| 1회 반복 | O(n × K × d) |
| 전체 | O(i × n × K × d) |

n = 데이터 수, K = 클러스터 수, d = 차원, i = 반복 횟수

## 실무에서의 활용

### 1. 고객 세그멘테이션
구매 패턴으로 고객을 그룹화: "충성 고객", "가격 민감 고객", "신규 고객" 등.
마케팅 전략을 그룹별로 차별화.

### 2. 이미지 압축 (색상 양자화)
이미지의 수백만 색상을 K개로 줄임.
K=16이면 각 픽셀을 4비트로 표현 가능.

### 3. 문서 클러스터링
뉴스 기사를 주제별로 자동 분류: 정치, 경제, 스포츠 등.
단어 벡터(TF-IDF, 임베딩)에 K-Means 적용.

### 4. 이상치 탐지
어떤 클러스터에도 속하지 않는 점 = 이상치.
금융 사기 탐지, 네트워크 침입 탐지.

### 5. 벡터 양자화 (AI 모델 압축)
임베딩 벡터를 K개의 대표 벡터로 압축.
LLM의 모델 크기를 줄이는 기법 중 하나.
